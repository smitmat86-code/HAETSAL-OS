# Implementation Plan: Phase 2.1 — Queue Topology + Ingestion Foundation

## Goal

Build the ingestion nervous system: Telnyx SMS webhook, salience scoring, dedup, write policy enforcement, TMK-encrypted Hindsight retain calls, and STONE archival. Replace `brain_v1_retain` stub with real pipeline. Every incoming artifact flows through a single `retainContent()` path.

## User Review Required

> [!IMPORTANT]
> **Telnyx Ed25519 verification library.** The spec calls for Ed25519 signature validation on the SMS webhook. Cloudflare Workers' `crypto.subtle` supports Ed25519 verification natively (via the Web Crypto API). Confirm no additional npm package is needed — if the Telnyx webhook signing format requires library-specific parsing (like `telnyx` npm package), we will need to pin that dependency.

> [!IMPORTANT]
> **Hindsight retain API contract.** The spec documents a `POST /api/retain` contract. Before implementation, we should verify this against the actual Hindsight v0.4.16 source (pinned at `58fdac4`). If the API shape differs, the `HindsightRetainRequest` type will need adjustment — logged as a spec deviation.

> [!NOTE]
> **Migration numbering confirmed.** Actual `migrations/` directory contains `1001–1004`. The `1005_brain_observability.sql` reference in CONVENTIONS.md is a documentation typo (observability is `1002`). **1005 is the next available number.** Numbering: 1005 (ingestion/2.1), 1006 (google/2.2), 1007 (bootstrap/2.4).

> [!IMPORTANT]
> **Step 0 — Pre-implementation verification.** Before writing any code, the implementing agent must run `ls migrations/` and confirm the next available migration number. Do not trust documentation — trust the filesystem.

### Core Ingestion Services

#### [NEW] [dedup.ts](file:///c:/Users/matth/Documents/HAETSAL%20OS/src/services/ingestion/dedup.ts)
- `computeDedupHash(source, content)` — SHA-256 of `source + normalized_content`
- `checkDedup(hash, tenantId, env)` — D1 `SELECT 1` on `ingestion_events.dedup_hash`
- Normalization: whitespace collapse, lowercase
- ~50 lines

#### [NEW] [salience.ts](file:///c:/Users/matth/Documents/HAETSAL%20OS/src/services/ingestion/salience.ts)
- `scoreSalience(artifact)` → `SalienceResult`
- Tier 3: explicit retain, self-SMS, decision language + date, surprise >0.7
- Tier 2: named person + context, meetings, financial, known contact SMS
- Tier 1: routine/no entities/no decisions
- Queue routing: Tier 2–3 → `QUEUE_HIGH`, Tier 1 → `QUEUE_NORMAL`
- Surprise score stub: always `0.5` (Phase 3 refines)
- ~80 lines

#### [NEW] [domain.ts](file:///c:/Users/matth/Documents/HAETSAL%20OS/src/services/ingestion/domain.ts)
- `inferDomain(content)` — keyword → domain mapping
- `inferMemoryType(content)` — defaults to `'episodic'` for ingestion content
- DOMAIN_KEYWORDS map: career, health, finance, relationships, etc.
- Default domain: `'general'`
- ~50 lines

#### [NEW] [write-policy.ts](file:///c:/Users/matth/Documents/HAETSAL%20OS/src/services/ingestion/write-policy.ts)
- `runWritePolicyValidator(content, inferredMemoryType, env)` — two-stage
- Stage 1: regex heuristic (`PROCEDURAL_PATTERNS` — always/never/every time etc.)
- Stage 2: Workers AI classifier via AI Gateway (only if heuristic flags)
- Short-circuit: explicit `memory_type: 'procedural'` → immediate drop
- Returns `{ isProcedural, method }` for audit context
- ~80 lines

#### [NEW] [retain.ts](file:///c:/Users/matth/Documents/HAETSAL%20OS/src/services/ingestion/retain.ts)
- `retainContent(artifact, tmk, env)` — **the single path for all memory writes**
- Pipeline: dedup → write policy → encrypt (AES-256-GCM) → R2 STONE → Hindsight retain → D1 batch (ingestion_events + memory_audit)
- Returns `RetainResult | null` (null = dedup hit or write policy violation)
- R2 STONE write via `waitUntil()` (non-blocking)
- D1 batch: atomic `INSERT OR IGNORE` on ingestion_events + `INSERT` on memory_audit
- ~120 lines

> - Main `index.ts` imports and mounts these as Hono route groups
> - Queue consumer dispatched to `src/workers/ingestion/consumer.ts`
>
> This prevents 2.2 from needing to restructure what 2.1 built.
