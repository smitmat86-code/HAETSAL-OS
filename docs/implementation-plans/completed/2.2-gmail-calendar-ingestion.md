# Implementation Plan: Phase 2.2 — Gmail + Calendar Ingestion

## Goal

Wire Gmail threads, Calendar events, and Obsidian notes as ingestion sources. All three funnel into the `retainContent()` pipeline from 2.1. Replace `brain_v1_recall` stub with real Hindsight recall. Add Google OAuth token management (encrypted in KV).

## User Review Required

> [!IMPORTANT]
> **Google API scopes.** The spec uses `gmail.readonly`, `calendar.readonly`, and `drive.readonly`. Confirm these are the minimal scopes needed. Gmail Push Notifications also require `pubsub.readonly` for the Google Cloud Pub/Sub topic setup — this may need a manual dashboard configuration step before webhooks work.

> [!IMPORTANT]
> **OAuth callback route.** `GET /auth/google/callback` processes the OAuth code exchange. This route needs to be behind CF Access (the user initiates the flow from the Pages UI). However, Google redirects to this URL — confirm that CF Access will allow the redirect without re-prompting auth (the browser already has a CF Access session cookie).

> [!WARNING]
> **Obsidian polling relies on Google Drive API.** The cron trigger polls Google Drive for `/to-brain/` files and `brain: true` frontmatter notes. This requires Drive API access configured per-tenant. If a tenant hasn't connected Google Drive, the cron should skip gracefully (not error).
