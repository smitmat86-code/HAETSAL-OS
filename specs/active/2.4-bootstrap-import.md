## What We're Building

The bootstrap that transforms THE Brain from an empty system to a populated one.
Two parts run in sequence as a durable Cloudflare Workflow:

**Phase A ‚Äî Intake Interview:** The Chief of Staff agent (in the McpAgentDO,
over an MCP session) conducts a structured interview domain-by-domain. Questions
like "What are your top 3 career goals right now?", "What health habits matter
most to you?". User answers are retained as `memory_type: 'semantic'`,
`provenance: 'user_authored'`. This is the highest-trust content in the Brain.

**Phase B ‚Äî Historical Import:** After the interview (or in parallel, tenantcontrolled), import Gmail (12 months), Calendar (24 months), and Google Drive
docs (3 years). Content is date-weighted: older content gets lower salience.
All on QUEUE_BULK ‚Äî never contends with real-time ingestion.

**Phase C ‚Äî Handoff:** Workflow completes, writes a bootstrap summary to Hindsight,
updates tenant bootstrap status in D1, and sends a WebSocket push to Pages UI.

When complete: a new tenant can bootstrap from scratch to a populated Brain in a
single guided session. The experience: MCP client opens, Chief of Staff asks
questions, user answers, background import runs asynchronously.

## THE Brain Laws Check

### Law 1: One Public Face
No new public routes. Bootstrap is triggered:
- Via MCP tool `brain_v1_bootstrap_start` (new ‚Äî registers on McpAgentDO)
- Via Pages UI button (1.4 wires the button ‚Äî this session wires the MCP tool)

The Workflow itself has no public HTTP surface.

### Law 2: Zero-Knowledge
- All interview answers encrypted with TMK before Hindsight retain
- All imported Gmail/Calendar content follows 2.1/2.2 encryption path
- Bootstrap progress stored in D1 as metadata only (phase, counts, timestamps)
- Workflow instance ID stored in `tenants.bootstrap_workflow_id` (new column via
  migration 1007)

### Law 3: Agent Write Policy
Chief of Staff writes `memory_type: 'semantic'` for structured interview answers
(facts about the tenant's values, goals, priorities ‚Äî semantic, not procedural).
Historical import writes `memory_type: 'episodic'` for events.
Write policy validator runs on ALL retain calls ‚Äî Chief of Staff must not sneak
procedural memories in through interview answers.

**Cloudflare Workflow phases:**
```typescript
// src/workflows/bootstrap.ts
export class BootstrapWorkflow extends WorkflowEntrypoint<Env, BootstrapParams> {
  async run(event: WorkflowEvent<BootstrapParams>, step: WorkflowStep) {

    // Phase A: Interview (waits for interview completion signal)
    const interviewDone = await step.do('await-interview', async () => {
      // Poll D1 for interview_completed_at ‚Äî Workflow sleeps between polls
      // Chief of Staff runs in MCP session (not here)
      // This step resolves when MCP session writes interview_completed_at
      return await this.waitForInterview(event.payload.tenantId)
    })

    // Phase B: Historical import (durable ‚Äî survives Worker restarts)
    await step.do('import-gmail', async () => {
      await this.importGmail(event.payload.tenantId, 12) // 12 months
    })
    await step.do('import-calendar', async () => {
      await this.importCalendar(event.payload.tenantId, 24) // 24 months
    })
    await step.do('import-drive', async () => {
      await this.importDrive(event.payload.tenantId, 36) // 36 months
    })

    // Phase C: Handoff
    await step.do('bootstrap-complete', async () => {
      await this.completeBootstrap(event.payload.tenantId)
    })
  }
}
```

**Date-weighted salience for historical import:**
```typescript
// Older content = lower salience
// Decay function: salience_multiplier = 1.0 - (age_months / max_months) * 0.5
// Minimum multiplier: 0.5 (old content still has value)
// Result mapped to tier: multiplier > 0.8 ‚Üí Tier 2, else ‚Üí Tier 1

function historicalSalienceMultiplier(occurredAt: number, maxMonths: number): number {
  const ageMs = Date.now() - occurredAt
  const ageMonths = ageMs / (1000 * 60 * 60 * 24 * 30)
  return Math.max(0.5, 1.0 - (ageMonths / maxMonths) * 0.5)
}
```

**`provenance = 'user_authored'` vs `provenance = 'bootstrap_import'`:**
- `user_authored`: Interview answers (highest trust ‚Äî direct from user)
- `bootstrap_import`: Gmail/Calendar/Drive historical import (lower trust)
- `user_authored` memories are never overwritten by consolidation cron
- `bootstrap_import` memories can be superseded by later evidence

## Part 3: Key Service Patterns

```typescript
// src/services/bootstrap/historical-import.ts (~120 lines)

export async function importGmailHistory(
  tenantId: string,
  hindsightTenantId: string,
  monthsBack: number,
  accessToken: string,
  env: Env
): Promise<number> {
  const since = Date.now() - monthsBack * 30 * 24 * 60 * 60 * 1000
  let imported = 0
  let pageToken: string | undefined

  do {
    const threads = await listGmailThreads(accessToken, since, pageToken)

    // Batch to QUEUE_BULK in groups of 50
    const batch = threads.items.map(thread => ({
      type: 'bootstrap_gmail_thread',
      tenant_id: tenantId,
      hindsight_tenant_id: hindsightTenantId,
      thread_id: thread.id,
      access_token_kv_key: `oauth:${tenantId}:google:gmail.readonly`,
      occurred_at: thread.internalDate,
      salience_multiplier: historicalSalienceMultiplier(thread.internalDate, monthsBack),
    }))

    // Send in sub-batches of 50
    for (let i = 0; i < batch.length; i += 50) {
      await env.QUEUE_BULK.sendBatch(
        batch.slice(i, i + 50).map(msg => ({ body: msg }))
      )
    }

    imported += batch.length
    pageToken = threads.nextPageToken
  } while (pageToken)

  return imported
}

// src/services/bootstrap/interview.ts (~100 lines)

export class InterviewSession {
  private domainIndex = 0
  private questionIndex = 0
  private answers: Array<{ domain: string; question: string; answer: string }> = []

  currentQuestion(): string | null {
    const domain = INTERVIEW_DOMAINS[this.domainIndex]
    if (!domain) return null  // Interview complete
    return domain.questions[this.questionIndex]
  }

  async recordAnswer(
    answer: string,
    tenantId: string,
    hindsightTenantId: string,
    tmk: CryptoKey,
    env: Env
  ): Promise<{ nextQuestion: string | null; complete: boolean }> {
    const domain = INTERVIEW_DOMAINS[this.domainIndex]
    if (!domain) return { nextQuestion: null, complete: true }

    this.answers.push({ domain: domain.domain, question: domain.questions[this.questionIndex], answer })

    // Retain answer as semantic memory
    await retainContent({
      source: 'mcp_retain',
      tenantId,
      hindsightTenantId,
      rawContent: `Q: ${domain.questions[this.questionIndex]}\nA: ${answer}`,
      occurredAt: Date.now(),
      provenance: 'user_authored',
      memoryType: 'semantic',
      domain: domain.domain,
      metadata: { bootstrap_interview: 'true', salience_tier: '3' }
    }, tmk, env)

    // Advance question
    this.questionIndex++
    if (this.questionIndex >= domain.questions.length) {
      this.domainIndex++
      this.questionIndex = 0
    }

    const nextQuestion = this.currentQuestion()
    return { nextQuestion, complete: nextQuestion === null }
  }
}
```
