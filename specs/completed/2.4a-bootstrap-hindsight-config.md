# Spec: Bootstrap Hindsight Configuration Addendum

**Phase:** 2 | **Session:** 2.4a | **Priority:** P0 (blocks 3.3 operational)
**Addendum to:** `2.4-bootstrap-import.md`
**Commit base:** Post-3.3 (`5eb957d`)

**What this is:** 3 Hindsight API calls that must fire at the end of Phase C
(bootstrap handoff). They were not in the original 2.4 spec — identified during
the Hindsight v0.4.16 audit that produced the revised 3.3 spec.

**Why P0:** Until these run:
- Hindsight's native fact consolidation fires but has no steering (`observations_mission`)
  — it extracts arbitrary facts rather than domain-relevant ones
- `BaseAgent.open()` calls `GET /mental-models/mental-model-{domain}` and gets 404
  for every domain — agents open with no mental model context
- The `consolidation.completed` webhook is not registered — 3.3's webhook-first
  trigger never fires, cron fallback only

**Depends on:**
- `2.4` as-built: `BootstrapWorkflow.completeBootstrap()` confirmed entry point
  for Phase C additions. `hindsightBankId` available in Workflow context.
- `3.3` as-built: `/hindsight/webhook` route confirmed at Worker URL.
  `HINDSIGHT_WEBHOOK_SECRET` confirmed in env.ts and wrangler secrets.
- Hindsight v0.4.16 API: `PUT /banks/{id}`, `POST /banks/{id}/mental-models`,
  `POST /banks/{id}/webhooks` all confirmed from API audit.

## What We're Adding

Three calls inside `completeBootstrap()` in `src/workflows/bootstrap.ts`,
wrapped in their own `step.do()` for durability. If any fail, Workflow retries
without re-running the import phases.

### 1. Bank Configuration (`observations_mission` + `retain_mission`)

Steers Hindsight's automatic fact extraction and observation synthesis to focus
on domain-relevant content rather than generic facts.

```typescript
await step.do('configure-hindsight-bank', async () => {
  const res = await env.HINDSIGHT.fetch(
    `/v1/default/banks/${hindsightBankId}`,
    {
      method: 'PUT',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        retain_mission: `This is a personal AI second brain for a professional.
          During fact extraction, prioritize:
          - Career decisions, professional milestones, and work relationships
          - Health habits, fitness patterns, and wellness commitments
          - Personal relationships and social dynamics
          - Learning moments, skills acquired, and intellectual growth
          - Financial decisions, goals, and constraints
          - Faith, values, and personal principles
          Deprioritize: logistics, scheduling details, generic pleasantries,
          one-off administrative tasks.`,

        observations_mission: `Synthesize durable facts about this person's
          professional trajectory, key relationships, recurring behavioral
          tendencies, and domain expertise. Observations should be stable
          claims that will still be accurate in 6 months.
          Examples of good observations:
          - "Tends to take on leadership roles in ambiguous situations"
          - "Has a recurring tension between career ambition and work-life balance"
          - "Closest professional relationships are with [entities from memories]"
          Examples of bad observations (too transient):
          - "Had a meeting on Tuesday"
          - "Sent an email about project X"
          Ignore one-off events. Focus on patterns and durable facts.`,

        enable_observations: true,
      })
    }
  )
  if (!res.ok) throw new Error(`Bank config failed: ${res.status}`)
})
```

### 2. Mental Models Per Domain (6 total)

Creates one Mental Model per domain with `refresh_after_consolidation: true`.
Hindsight refreshes these automatically after every `/consolidate` call —
which is what replaces Pass 5 from the original 3.3 spec.

`BaseAgent.open()` fetches these by stable ID: `mental-model-{domain}`.

```typescript
await step.do('create-mental-models', async () => {
  const DOMAINS = [
    {
      id: 'career',
      name: 'Career & Professional',
      query: `What are the key patterns, goals, current challenges, and
        trajectory in this person's career and professional life?
        Include: role, ambitions, key relationships, recurring themes,
        significant decisions made. Focus on durable facts and direction.`
    },
    {
      id: 'health',
      name: 'Health & Wellness',
      query: `What are this person's health habits, fitness patterns,
        wellness commitments, and physical/mental health context?
        Include: routines maintained, goals, challenges, trajectory.`
    },
    {
      id: 'relationships',
      name: 'Relationships & Social',
      query: `Who are the most important people in this person's life,
        and what is the nature of those relationships?
        Include: family, close friends, mentors, key professional contacts.
        Focus on relationship quality and dynamics, not just names.`
    },
    {
      id: 'learning',
      name: 'Learning & Growth',
      query: `What is this person actively learning, curious about,
        or developing as a skill or capability?
        Include: formal learning, self-directed study, intellectual interests,
        skills being built, books/content consumed with impact.`
    },
    {
      id: 'finance',
      name: 'Financial',
      query: `What are this person's financial goals, constraints,
        significant decisions, and overall financial trajectory?
        Focus on goals and patterns — not specific account details.`
    },
    {
      id: 'general',
      name: 'General & Cross-Domain',
      query: `What are the overarching themes, values, and priorities
        that cut across all areas of this person's life?
        Include: core values stated or demonstrated, life philosophy,
        recurring tensions or trade-offs, what they're optimizing for.`
    },
  ]

  const results = await Promise.allSettled(
    DOMAINS.map(domain =>
      env.HINDSIGHT.fetch(
        `/v1/default/banks/${hindsightBankId}/mental-models`,
        {
          method: 'POST',
          headers: { 'Content-Type': 'application/json' },
          body: JSON.stringify({
            id: `mental-model-${domain.id}`,  // stable ID — BaseAgent.open() uses this
            name: domain.name,
            source_query: domain.query,
            tags: [`domain:${domain.id}`],
            max_tokens: 2048,
            trigger: { refresh_after_consolidation: true }
          })
        }
      ).then(res => {
        if (!res.ok && res.status !== 409) {  // 409 = already exists, idempotent
          throw new Error(`Mental model ${domain.id} failed: ${res.status}`)
        }
      })
    )
  )

  // Log failures but don't throw — partial success is acceptable
  // BaseAgent.open() handles 404 gracefully (opens with no mental model context)
  const failures = results.filter(r => r.status === 'rejected')
  if (failures.length > 0) {
    console.error(`Mental model creation: ${failures.length}/6 failed`, failures)
  }
})
```

### 3. Consolidation Webhook Registration

Registers THE Brain's Worker as a Hindsight webhook receiver for
`consolidation.completed` events. This is what makes 3.3 webhook-first
rather than cron-only.

```typescript
await step.do('register-consolidation-webhook', async () => {
  // Check if webhook already registered (idempotent re-run)
  const listRes = await env.HINDSIGHT.fetch(
    `/v1/default/banks/${hindsightBankId}/webhooks`
  )
  const { items } = await listRes.json()
  const alreadyRegistered = items.some(
    (w: any) => w.url.includes('/hindsight/webhook') && w.enabled
  )
  if (alreadyRegistered) return  // idempotent

  const res = await env.HINDSIGHT.fetch(
    `/v1/default/banks/${hindsightBankId}/webhooks`,
    {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        url: `https://${env.WORKER_DOMAIN}/hindsight/webhook`,
        secret: env.HINDSIGHT_WEBHOOK_SECRET,
        event_types: ['consolidation.completed'],
        enabled: true,
      })
    }
  )
  if (!res.ok) throw new Error(`Webhook registration failed: ${res.status}`)
})
```

## Source Modifications

### [MODIFY] `src/workflows/bootstrap.ts`

Insert 3 new `step.do()` calls inside `completeBootstrap()`, **before** the
existing WebSocket push and `tenants.bootstrap_status = 'completed'` update.

Order matters:
```
1. configure-hindsight-bank       (observations_mission, retain_mission)
2. create-mental-models           (6 domains, refresh_after_consolidation)
3. register-consolidation-webhook (consolidation.completed → /hindsight/webhook)
4. [existing] bootstrap-complete  (D1 status update, WebSocket push)
```

Steps 1-3 run inside the existing `step.do('bootstrap-complete')`, OR as
separate named steps if `completeBootstrap()` is already a single `step.do()`.
Prefer separate named steps for granular retry — if webhook registration fails,
Workflow retries only that step, not bank config or mental model creation.

### [NEW] `scripts/backfill-hindsight-config.ts` (~80 lines)

```typescript
// scripts/backfill-hindsight-config.ts
// Run: npx tsx scripts/backfill-hindsight-config.ts
//
// Applies the 2.4a bootstrap addendum to an already-bootstrapped tenant.
// Safe to re-run — all operations are idempotent.

import { config } from 'dotenv'
config({ path: '.dev.vars' })

const HINDSIGHT_URL = process.env.HINDSIGHT_URL!
const HINDSIGHT_BANK_ID = process.env.HINDSIGHT_BANK_ID!
const HINDSIGHT_WEBHOOK_SECRET = process.env.HINDSIGHT_WEBHOOK_SECRET!
const WORKER_DOMAIN = process.env.WORKER_DOMAIN!

async function hindsight(path: string, opts?: RequestInit) {
  const res = await fetch(`${HINDSIGHT_URL}${path}`, {
    ...opts,
    headers: {
      'Content-Type': 'application/json',
      ...(opts?.headers ?? {}),
    },
  })
  if (!res.ok && res.status !== 409) {
    const body = await res.text()
    throw new Error(`Hindsight ${path} → ${res.status}: ${body}`)
  }
  return res
}

async function main() {
  console.log('Step 1: Configuring bank (observations_mission + retain_mission)...')
  await hindsight(`/v1/default/banks/${HINDSIGHT_BANK_ID}`, {
    method: 'PUT',
    body: JSON.stringify({
      retain_mission: `Focus on: career decisions and professional milestones,
        health patterns and habits, relationship dynamics, learning and growth
        moments, financial decisions, faith and values. Ignore: logistics,
        scheduling details, generic pleasantries.`,
      observations_mission: `Synthesize durable facts about professional
        trajectory, key relationships, recurring behavioral patterns, and
        domain expertise. Observations should be stable claims accurate in
        6 months. Ignore one-off events. Focus on patterns and durable facts.`,
      enable_observations: true,
    }),
  })
  console.log('✓ Bank configured')

  console.log('Step 2: Creating mental models (6 domains)...')
  const DOMAINS = ['career', 'health', 'relationships', 'learning', 'finance', 'general']
  const QUERIES: Record<string, string> = {
    career: 'Key patterns, goals, challenges, and trajectory in career and professional life?',
    health: 'Health habits, fitness patterns, wellness commitments, and trajectory?',
    relationships: 'Most important people, relationship quality, and social dynamics?',
    learning: 'Active learning, intellectual interests, skills being built?',
    finance: 'Financial goals, constraints, significant decisions, trajectory?',
    general: 'Overarching themes, values, priorities, and what this person optimizes for?',
  }

  for (const domain of DOMAINS) {
    await hindsight(`/v1/default/banks/${HINDSIGHT_BANK_ID}/mental-models`, {
      method: 'POST',
      body: JSON.stringify({
        id: `mental-model-${domain}`,
        name: `${domain.charAt(0).toUpperCase() + domain.slice(1)} Mental Model`,
        source_query: QUERIES[domain],
        tags: [`domain:${domain}`],
        max_tokens: 2048,
        trigger: { refresh_after_consolidation: true },
      }),
    })
    console.log(`  ✓ mental-model-${domain}`)
  }

  console.log('Step 3: Registering consolidation webhook...')
  const listRes = await hindsight(`/v1/default/banks/${HINDSIGHT_BANK_ID}/webhooks`)
  const { items } = await listRes.json() as any
  const exists = items.some((w: any) => w.url.includes('/hindsight/webhook') && w.enabled)

  if (exists) {
    console.log('  ✓ Webhook already registered (skipped)')
  } else {
    await hindsight(`/v1/default/banks/${HINDSIGHT_BANK_ID}/webhooks`, {
      method: 'POST',
      body: JSON.stringify({
        url: `https://${WORKER_DOMAIN}/hindsight/webhook`,
        secret: HINDSIGHT_WEBHOOK_SECRET,
        event_types: ['consolidation.completed'],
        enabled: true,
      }),
    })
    console.log('  ✓ Webhook registered')
  }

  console.log('\n✅ 2.4a backfill complete. THE Brain is now fully operational.')
  console.log('   - Hindsight will extract domain-relevant facts on next retain')
  console.log('   - Mental models will auto-refresh after next consolidation')
  console.log('   - Webhook will trigger 3.3 cron on next consolidation.completed event')
}

main().catch(e => { console.error('❌ Backfill failed:', e); process.exit(1) })
```

**Required `.dev.vars` entries for backfill:**
```
HINDSIGHT_URL=http://localhost:8888
HINDSIGHT_BANK_ID=<your-hindsight-bank-id>
HINDSIGHT_WEBHOOK_SECRET=<from-wrangler-secrets>
WORKER_DOMAIN=<your-worker.workers.dev>
```

## Constraints

### Must Use
- `step.do()` for each of the 3 Hindsight calls — durable retry if one fails
- `Promise.allSettled()` for the 6 parallel mental model creates
- 409 handling on mental model create — idempotent re-run safe
- Existence check before webhook registration — no duplicates

### Must NOT
- Hardcode Worker URL — always use `env.WORKER_DOMAIN`
- Throw on partial mental model failure — log and continue
- Block bootstrap completion if Hindsight config fails after 3 retries
  (log anomaly_signal, mark bootstrap completed anyway — Brain works without
  perfect mental models, just less context at `open()`)
