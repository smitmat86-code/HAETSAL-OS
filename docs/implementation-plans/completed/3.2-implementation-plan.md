# Spec 3.2 — Career Coach (First Domain Agent)

## Goal

Build the first concrete domain agent on the `BaseAgent` foundation. Career Coach loads career-specific memories, writes structured session synthesis, and establishes the delegation + memory interface patterns that all future domain agents will follow.

### Career Coach Agent

#### [NEW] [career-coach.ts](file:///c:/Users/matth/Documents/HAETSAL%20OS/src/agents/career-coach.ts)

Concrete BaseAgent subclass (~120 lines):

- `domain = 'career'`, `agentIdentity = 'career_coach'`
- **`open()`** — calls `super.open()`, then 2 additional Hindsight recall calls:
  1. Professional relationships (semantic, domain='career', limit=8)
  2. Recent career decisions (episodic, domain='career', limit=5)
  - Results decrypted and added to `this.context.careerRelationships` / `this.context.recentDecisions`
- **System prompt** — career-specific: references loaded goals, relationships, pending career actions. Explicitly states: "You do NOT write personality rules or behavioral patterns — the nightly process does that"
- **`close()`** — builds structured synthesis (`buildCareerSynthesis()`) with topics + proposed actions + key insight → `this.retain(synthesis, 'episodic', 'career', 'career_coach_session')` → `super.close(synthesis)`
- **`buildCareerSynthesis()`** — private helper: timestamp, topics list, proposed actions list, key insight. Feeds nightly cron Pass 4
- **`getCareerContext()`** — returns formatted string of career goals + relationships for MCP tool

### McpAgent.ts — Memory Interface + Delegation

#### [MODIFY] [McpAgent.ts](file:///c:/Users/matth/Documents/HAETSAL%20OS/src/workers/mcpagent/do/McpAgent.ts)

Register 4 new things (~40 lines added, watch 200 line limit):

**1. `memory_search` tool** — available in ALL sessions:
```typescript
this.server.tool('memory_search', 'Search memories', {
  query: z.string(),
  domain: z.string().optional(),
  limit: z.number().optional(),
}, async ({ query, domain, limit }) => {
  // Recall from Hindsight → decrypt with TMK → return plaintext
})
```

**2. `memory_write` tool** — available in ALL sessions:
```typescript
this.server.tool('memory_write', 'Write a memory', {
  content: z.string(),
  memory_type: z.enum(['episodic', 'semantic']),  // No 'procedural' — Zod enforces
  domain: z.string().optional(),
}, async ({ content, memory_type, domain }) => {
  // retainContent() → return { memory_id }
})
```

> [!IMPORTANT]
> `memory_write` Zod schema uses `z.enum(['episodic', 'semantic'])` — **deliberately excluding both `'procedural'` and `'world'`**.
> - **`procedural` excluded**: Law 3 — only the consolidation cron writes behavioral patterns.
> - **`world` excluded**: `world` represents external/public knowledge (news, market data, facts about the world). It enters through the ingestion pipeline or cron-synthesized sources — not through MCP callers self-classifying. Tenants write personal observations (`episodic`) or learned facts (`semantic`).
> - Agent-level enforcement is the type system (`EpistemicMemoryType`); MCP tool-level enforcement is Zod.

**3. `career_context` tool** — session-scoped (only when Career Coach active)

**4. `career_note` tool** — session-scoped (only when Career Coach active)

**5. `instantiateAgent(agentType)`** — factory: maps `AgentType` → agent class

**6. `handleDelegation(signal: DelegationSignal)`** — instantiate agent, run with parent trace
