## What We're Building

The nervous system that routes all incoming information into THE Brain. An Ingestion
Worker accepts webhooks from sources (Telnyx SMS in this session) and internal queue
messages, runs each artifact through the salience scorer, deduplicates against D1,
encrypts content with the tenant TMK, and calls Hindsight's retain API. The write
policy validator (heuristic + Workers AI classifier) wires here — replacing the
`// TODO: Phase 2.1` stub in `brain_v1_retain`.

When complete: an inbound SMS via Telnyx routes through the pipeline, gets salience-
scored, deduplicated, encrypted, retained in Hindsight, and produces an audit record
in D1. `brain_v1_retain` from an MCP session does the same. Both paths share the
same core retain service.

## THE Brain Laws Check

### Law 1: One Public Face
The Ingestion Worker adds one new public route: `POST /ingest/sms` for the Telnyx
webhook. This route must be authenticated — Telnyx signs webhooks with HMAC-SHA256.
Validate the `telnyx-signature-ed25519` header before processing. This is NOT behind
CF Access (Telnyx can't do CF Access OAuth) — it uses Telnyx webhook signature
verification instead. Document this exception explicitly in the route handler.

All other ingestion paths are internal queue consumers — no public surface.

### Law 2: Zero-Knowledge
This session performs THE Brain's first real memory writes. The encryption contract:
1. Ingestion Worker receives plaintext artifact
2. Fetch tenant TMK from McpAgentDO via DO RPC (`getTmk(tenantId)`) — new RPC method
3. Encrypt content AES-256-GCM with TMK before any Hindsight call
4. Send encrypted bytes + IV to Hindsight retain API
5. Hindsight stores ciphertext only — never sees plaintext
6. D1 `ingestion_events` stores: dedup_hash, salience_tier, source, r2_key — never content
7. `memory_audit` stores: operation, memory_type, domain, salience_tier — never content

**TMK access pattern for Ingestion Worker:** The TMK lives in McpAgentDO memory.
The Ingestion Worker does not have direct access. Pattern:
```
Ingestion Worker → env.MCPAGENT.idFromName(tenantId) → stub.getTmk() → CryptoKey
```
`getTmk()` is a new DO RPC method added to McpAgentDO in this session. It returns
the TMK only if the DO is warm (tenant has an active session). If DO is cold (TMK
not yet derived), return `null` — queue the artifact for retry via QUEUE_NORMAL
with a 30-second delay. Never block ingestion on a cold DO.

**R2 raw artifact storage:** Before encryption and Hindsight retain, store the raw
artifact in R2_ARTIFACTS at `artifacts/{tenantId}/{source}/{dedup_hash}`. This is
the STONE (Source of Truth, Original, Never-Edited) record. The R2 object stores
plaintext — it is in a customer-controlled bucket. Document this in the spec.
Re-extraction (Phase 4.1) reads from R2, not from Hindsight.

### Law 3: Agent Write Policy
The write policy validator wires here. Two-stage:
1. **Heuristic check** (~5ms): regex patterns for sweeping generalizations
   (`always`, `never`, `everyone`, `no one`) that suggest procedural memory.
   If flagged → route to classifier.
2. **Workers AI classifier** (~200ms): only if heuristic flags. Binary:
   `episodic/semantic/world` vs `procedural`. If `procedural` → silent drop +
   `memory.write_policy_violation` anomaly signal. Return success to caller
   (doom loop prevention).

The validator runs on ALL retain paths: `brain_v1_retain` MCP tool AND ingestion
pipeline. Ingestion content is less likely to be procedural, but the validator
runs unconditionally.

## Behavioral Wiring

### Authorization

| Route / Entry Point | Auth | Tenant ID Source | Notes |
|---------------------|------|-----------------|-------|
| `POST /ingest/sms` | Telnyx Ed25519 signature | Telnyx `to` number → tenant lookup | NOT CF Access — document exception |
| Queue consumer (QUEUE_HIGH/NORMAL/BULK) | Internal — tenantId from queue message | Queue message `tenant_id` | No HTTP auth needed |
| `brain_v1_retain` (MCP tool) | CF Access via McpAgentDO | Already validated by 1.2 auth | Replaces stub — real Hindsight call |

**Telnyx phone → tenant mapping:** `tenant_phone_numbers` table (new in this
session). Maps E.164 phone number → tenant_id. Indexed on phone number.

### Action Layer
No new action tools in this session. `brain_v1_act_send_message` stub from 1.3
will use the real SMS send path (Phase 2.3+). Not wired here.

### Audit

| Event | Table | Action String | Blocking? |
|-------|-------|--------------|-----------|
| Artifact ingested + retained | `memory_audit` | `memory.retained` | `waitUntil()` |
| Dedup hit (already seen) | `ingestion_events` (no new row) | — | No audit — silent skip |
| Write policy violation (procedural) | `anomaly_signals` | — | `waitUntil()` |
| Telnyx signature invalid | `memory_audit` | `ingest.signature_invalid` | `waitUntil()` |
| TMK cold (DO warm miss) | `ingestion_events` | `ingest.tmk_cold_retry` | `waitUntil()` |
| Salience scored | Embedded in `ingestion_events` row | — | Atomic with INSERT |

### Compute Routing

| Operation | Tier | Rationale |
|-----------|------|-----------|
| Telnyx webhook receipt | Sync (C1) | <5ms — validate sig, enqueue, return 200 |
| Salience scoring (heuristic) | Sync in queue consumer | <10ms |
| Write policy heuristic check | Sync in queue consumer | <5ms |
| Workers AI classifier (if flagged) | Sync in queue consumer | ~200ms — acceptable |
| Hindsight retain call | Sync in queue consumer | <500ms target |
| R2 raw artifact write | `waitUntil()` | Non-blocking — STONE record |
| Audit writes | `waitUntil()` | Non-blocking |

**Queue routing by salience tier:**
- Tier 1 (low salience — grocery lists, routine) → QUEUE_NORMAL
- Tier 2 (medium — meetings, decisions, notable events) → QUEUE_HIGH
- Tier 3 (high — explicit memory requests, significant events) → QUEUE_HIGH
- SMS always → QUEUE_HIGH (real-time capture expectation)
- Bootstrap import → QUEUE_BULK only (never QUEUE_HIGH/NORMAL)

## Part 4: Core Services

```typescript
// src/services/ingestion/retain.ts (~120 lines)
// The single path for all memory writes. Called by:
//   - Queue consumer (ingestion pipeline)
//   - brain_v1_retain MCP tool (direct)
// Law 2: encrypts before Hindsight call. Law 3: write policy before encrypt.

import type { Env } from '../../types/env'
import type { IngestionArtifact, RetainResult } from '../../types/ingestion'
import { scoreSalience } from './salience'
import { computeDedupHash, checkDedup } from './dedup'
import { runWritePolicyValidator } from './write-policy'
import { inferDomain, inferMemoryType } from './domain'

export async function retainContent(
  artifact: IngestionArtifact,
  tmk: CryptoKey,
  env: Env
): Promise<RetainResult | null> {  // null = dedup hit or write policy violation

  // 1. Dedup check
  const dedupHash = computeDedupHash(artifact.source, artifact.rawContent)
  const alreadySeen = await checkDedup(dedupHash, artifact.tenantId, env)
  if (alreadySeen) return null

  // 2. Write policy validation (Law 3)
  const domain = artifact.domain ?? inferDomain(artifact.rawContent)
  const memoryType = artifact.memoryType ?? inferMemoryType(artifact.rawContent)
  const policyResult = await runWritePolicyValidator(artifact.rawContent, memoryType, env)

  if (policyResult.isProcedural) {
    // Silent drop — return null, write anomaly signal async
    env.D1_US && env.ctx?.waitUntil(writeWritePolicyViolation(artifact, env))
    return null
  }

  // 3. Encrypt with TMK (Law 2)
  const iv = crypto.getRandomValues(new Uint8Array(12))
  const encoded = new TextEncoder().encode(artifact.rawContent)
  const ciphertext = await crypto.subtle.encrypt({ name: 'AES-GCM', iv }, tmk, encoded)
  const contentEncrypted = btoa(
    String.fromCharCode(...new Uint8Array([...iv, ...new Uint8Array(ciphertext)]))
  )

  // 4. Store STONE record in R2 (plaintext — customer-controlled bucket)
  const r2Key = `artifacts/${artifact.tenantId}/${artifact.source}/${dedupHash}`
  env.ctx?.waitUntil(
    env.R2_ARTIFACTS.put(r2Key, artifact.rawContent, {
      httpMetadata: { contentType: 'text/plain' },
      customMetadata: { source: artifact.source, occurred_at: String(artifact.occurredAt) }
    })
  )

  // 5. Call Hindsight retain
  const salience = scoreSalience(artifact)
  const hindsightRes = await env.HINDSIGHT.fetch('http://internal/api/retain', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      tenant_id: artifact.hindsightTenantId,
      content_encrypted: contentEncrypted,
      memory_type: memoryType,
      domain,
      provenance: artifact.provenance,
      occurred_at: artifact.occurredAt,
      salience_tier: salience.tier,
      metadata: {
        source: artifact.source,
        word_count: artifact.rawContent.split(/\s+/).length,
        ...artifact.metadata,
      }
    })
  })

  if (!hindsightRes.ok) throw new Error(`Hindsight retain failed: ${hindsightRes.status}`)
  const { memory_id } = await hindsightRes.json() as { memory_id: string }

  // 6. Write D1 records (atomic batch — ingestion_events + memory_audit)
  const now = Date.now()
  const eventId = crypto.randomUUID()
  await env.D1_US.batch([
    env.D1_US.prepare(
      `INSERT OR IGNORE INTO ingestion_events
       (id, tenant_id, created_at, source, salience_tier, surprise_score,
        memory_id, r2_key, dedup_hash)
       VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?)`
    ).bind(eventId, artifact.tenantId, now, artifact.source,
           salience.tier, salience.surpriseScore, memory_id, r2Key, dedupHash),
    env.D1_US.prepare(
      `INSERT INTO memory_audit
       (id, tenant_id, created_at, operation, memory_id, memory_type,
        domain, provenance, salience_tier)
       VALUES (?, ?, ?, 'memory.retained', ?, ?, ?, ?, ?)`
    ).bind(crypto.randomUUID(), artifact.tenantId, now,
           memory_id, memoryType, domain, artifact.provenance, salience.tier),
  ])

  return { memoryId: memory_id, salienceTier: salience.tier, memoryType, domain, dedupHash }
}

// src/services/ingestion/write-policy.ts (~80 lines)
// Law 3 enforcement: heuristic first, Workers AI classifier only if flagged
// Pattern-first: ~60-80% of content resolved without LLM call

const PROCEDURAL_PATTERNS = [
  /\b(always|never|every time|whenever|i always|you always)\b/i,
  /\b(the rule is|my rule|my policy|i never|i always)\b/i,
  /\b(everyone|no one|nobody|all people|people always)\b/i,
]

export async function runWritePolicyValidator(
  content: string,
  inferredMemoryType: string,
  env: Env
): Promise<{ isProcedural: boolean; method: 'heuristic' | 'classifier' | 'none' }> {

  // Short-circuit: only episodic/semantic/world are valid — if already procedural, drop
  // (This shouldn't happen from domain agents, but MCP callers can pass explicit types)
  if (inferredMemoryType === 'procedural') {
    return { isProcedural: true, method: 'none' }
  }

  // Heuristic check — fast regex pass
  const flagged = PROCEDURAL_PATTERNS.some(p => p.test(content))
  if (!flagged) {
    return { isProcedural: false, method: 'heuristic' }
  }

  // Workers AI classifier — only for flagged content
  // Route through AI Gateway (brain-gateway)
  const result = await env.AI.run(
    '@cf/meta/llama-3.1-8b-instruct',
    {
      messages: [{
        role: 'user',
        content: `Classify this memory: is it a behavioral rule/policy (procedural) or a specific event/fact (episodic/semantic)?
Memory: "${content.slice(0, 500)}"
Answer with exactly one word: procedural OR episodic`
      }]
    },
    { gateway: { id: 'brain-gateway' } }
  ) as { response: string }

  const isProcedural = result.response.trim().toLowerCase().startsWith('procedural')
  return { isProcedural, method: 'classifier' }
}
```

## Part 5: SMS Ingestion Route

```typescript
// Addition to src/workers/mcpagent/index.ts

// Telnyx SMS webhook — NOT behind CF Access (Telnyx can't do OAuth)
// Authentication: Ed25519 webhook signature verification
// LAW 1 EXCEPTION: Documented — Telnyx webhook uses signature auth, not CF Access
app.post('/ingest/sms', async (c) => {
  // Verify Telnyx Ed25519 signature
  const signature = c.req.header('telnyx-signature-ed25519')
  const timestamp = c.req.header('telnyx-timestamp')
  const body = await c.req.text()

  const isValid = await verifyTelnyxSignature(signature, timestamp, body, c.env.TELNYX_PUBLIC_KEY)
  if (!isValid) {
    c.executionCtx.waitUntil(
      writeAuditLog(c.env, 'ingest.signature_invalid', null)
    )
    return c.json({ error: 'Invalid signature' }, 403)
  }

  const payload = JSON.parse(body)
  const toPhone = payload.data?.payload?.to?.[0]?.phone_number
  const fromPhone = payload.data?.payload?.from?.phone_number
  const text = payload.data?.payload?.text
  const occurredAt = new Date(payload.data?.payload?.received_at ?? Date.now()).getTime()

  // Look up tenant by phone number
  const phoneRow = await c.env.D1_US.prepare(
    'SELECT tenant_id FROM tenant_phone_numbers WHERE phone_e164 = ?'
  ).bind(toPhone).first<{ tenant_id: string }>()

  if (!phoneRow) return c.json({ received: true }, 200)  // Unknown number — ignore silently

  // Enqueue for async processing — return 200 immediately (Telnyx requires fast ack)
  await c.env.QUEUE_HIGH.send({
    type: 'sms_inbound',
    tenant_id: phoneRow.tenant_id,
    from_phone: fromPhone,
    to_phone: toPhone,
    content: text,
    occurred_at: occurredAt,
    telnyx_event_id: payload.data?.id,
  })

  return c.json({ received: true }, 200)
})
```
