## What We're Building

Gmail threads, Calendar events, and Obsidian notes flowing into THE Brain. Three
distinct ingestion sources, each with different capture logic, but all funneling
into the same `retainContent()` pipeline from 2.1. Gmail uses Google Push
Notifications (webhooks) for real-time thread capture. Calendar uses the same
webhook mechanism for event updates. Obsidian syncs via Google Drive: a 1-minute
webhook watches `/to-brain/` folder drops, and a 15-minute cron polls for notes
with `brain: true` frontmatter anywhere in the vault.

When complete: receiving a Gmail thread (2+ replies) automatically retains a memory.
Creating or updating a calendar event retains a memory. Dropping a note into
`/to-brain/` in Obsidian retains a memory within ~1 minute. `brain_v1_recall` is
also upgraded to call Hindsight's real recall API (the stub is replaced).

## THE Brain Laws Check

### Law 1: One Public Face
Two new webhook routes — both on the existing McpAgent Worker (not new Workers):
- `POST /ingest/gmail` — Google Push Notification webhook
- `POST /ingest/calendar` — Google Calendar Push Notification webhook

Both use Google webhook signature verification (X-Goog-Channel-Token header with
pre-shared secret per tenant). Not behind CF Access — same documented exception
as Telnyx in 2.1. Document explicitly in route handler.

Obsidian ingestion is not a webhook — it's a cron + Drive API polling on a Worker
cron trigger. No new public surface.

### Law 2: Zero-Knowledge
Same encryption protocol as 2.1:
- Gmail thread body + metadata encrypted with tenant TMK before Hindsight retain
- Calendar event title + description encrypted; start/end times go into metadata
  (unencrypted — used for temporal graph edges, not sensitive)
- Obsidian note body encrypted; frontmatter metadata unencrypted
- Google OAuth tokens (access_token, refresh_token) encrypted with tenant TMK
  before storage in KV_SESSION. Never stored plaintext.
- `occurred_at` for Gmail: thread's first message date. Calendar: event start time.
  Obsidian: note `updated` timestamp from Drive.

### Law 3: Agent Write Policy
Write policy validator runs on all three ingestion paths (inherited from 2.1
`retainContent()` — no additional wiring needed). Calendar event titles are
unlikely to trigger procedural classification. Gmail threads with opinionated
language may be flagged — this is correct behavior.

## Behavioral Wiring

### Authorization

| Route / Entry Point | Auth | Tenant ID Source | Notes |
|---------------------|------|-----------------|-------|
| `POST /ingest/gmail` | Google channel token (pre-shared) | channel_token → tenant lookup | Not CF Access — documented |

**Google channel token → tenant mapping:**
New D1 table `google_webhook_channels` (see migration 1006). Maps `channel_id` +
`channel_token` → `tenant_id` + `resource_type` ('gmail' | 'calendar').

**Google OAuth token storage:**
New D1 table `google_oauth_tokens` stores encrypted token metadata. Actual
access/refresh tokens stored in KV_SESSION at key
`oauth:{tenantId}:google:{scope}` — encrypted with TMK.

OAuth setup flow (Pages UI in 1.4 wires the button — this session wires the
callback endpoint and token storage):
- `GET /auth/google/callback` — exchanges code for tokens, encrypts, stores in KV
- `POST /auth/google/revoke` — clears KV + D1 records

### Other Behavioral Wiring

**Gmail capture rules:**
- Threads with 2+ replies only (signal: conversation worth remembering)
- Single-message threads skipped (noise — too many promotional/transactional)
- Thread body: concatenate last 3 messages (most recent context)
- Trim to 2000 chars before encryption (Hindsight content limit advisory)
- Domain assignment: 'career' if work email domain, 'general' otherwise
- Provenance: `'gmail'`

**Calendar capture rules:**
- All events with duration > 15 minutes (skip quick 1:1 blocks)
- Capture: title + description + attendees count + duration + recurrence
- Attendees: count only, never names/emails (PII reduction)
- Domain: 'career' for work calendar, 'life' for personal
- `occurred_at`: event start time (not when it appeared on calendar)
- Provenance: `'calendar'`

**Obsidian ingestion rules:**
- `/to-brain/` folder: any `.md` file → QUEUE_HIGH, `salience_tier = 3` (explicit capture)
- Frontmatter `brain: true` anywhere in vault → QUEUE_NORMAL, `salience_tier = 2`
- Anti-pattern: filter out notes with `generated_by: the-brain` frontmatter
  (these are `/from-brain/` outputs — circular reference prevention)
- Parse `[[wikilinks]]` → candidate bridge edges (stored as metadata, not content)
  `metadata.wikilinks = ['ProjectAlpha', 'Q2Planning']`
- Provenance: `'obsidian'`

**Google OAuth token refresh:**
Access tokens expire after 1 hour. Queue consumers that need a Google API call
must check token expiry before calling. If expired: use refresh_token to get new
access_token, re-encrypt, store in KV. This happens inline in the queue consumer
(not a separate service) — add `refreshGoogleToken()` helper to `services/google/oauth.ts`.

**Wikilink bridge edges — not stored in D1:**
Wikilinks are passed as metadata to Hindsight. Hindsight creates the bridge edges
in the graph internally. D1 only stores the `ingestion_events` row. No new D1
tables needed for graph edges.

## Part 2: New Migration

```sql
-- migrations/1006_brain_google.sql

CREATE TABLE IF NOT EXISTS google_webhook_channels (
  id              TEXT PRIMARY KEY,     -- ulid
  tenant_id       TEXT NOT NULL,
  channel_id      TEXT NOT NULL UNIQUE, -- Google-assigned channel ID
  channel_token   TEXT NOT NULL,        -- pre-shared secret for verification
  resource_type   TEXT NOT NULL,        -- 'gmail' | 'calendar' | 'drive'
  resource_id     TEXT,                 -- mailbox ID or calendar ID
  expiry_at       INTEGER,              -- unix ms — Google channels expire
  created_at      INTEGER NOT NULL,
  FOREIGN KEY (tenant_id) REFERENCES tenants(id)
);

CREATE INDEX IF NOT EXISTS idx_google_channels_tenant
  ON google_webhook_channels(tenant_id, resource_type);

CREATE INDEX IF NOT EXISTS idx_google_channels_token
  ON google_webhook_channels(channel_token);

-- OAuth token metadata only — actual tokens in KV_SESSION (encrypted)
CREATE TABLE IF NOT EXISTS google_oauth_tokens (
  id              TEXT PRIMARY KEY,     -- ulid
  tenant_id       TEXT NOT NULL,
  scope           TEXT NOT NULL,        -- 'gmail.readonly' | 'calendar.readonly' | 'drive.readonly'
  kv_key          TEXT NOT NULL,        -- KV key where encrypted token lives
  expires_at      INTEGER,              -- access token expiry (unix ms)
  created_at      INTEGER NOT NULL,
  updated_at      INTEGER NOT NULL,
  FOREIGN KEY (tenant_id) REFERENCES tenants(id)
);

CREATE UNIQUE INDEX IF NOT EXISTS idx_google_oauth_unique
  ON google_oauth_tokens(tenant_id, scope);
```

## Part 3: Key Service Patterns

```typescript
// src/services/google/gmail.ts (~100 lines)

export async function fetchAndExtractThread(
  threadId: string,
  accessToken: string
): Promise<{ content: string; occurredAt: number; replyCount: number } | null> {
  const res = await fetch(
    `https://gmail.googleapis.com/gmail/v1/users/me/threads/${threadId}?format=full`,
    { headers: { Authorization: `Bearer ${accessToken}` } }
  )
  if (!res.ok) throw new Error(`Gmail API error: ${res.status}`)
  const thread = await res.json() as GoogleThread

  // Filter: 2+ replies only
  if (thread.messages.length < 2) return null

  // Extract last 3 messages, concatenate, trim to 2000 chars
  const relevant = thread.messages.slice(-3)
  const content = relevant
    .map(m => extractMessageBody(m))
    .join('\n\n---\n\n')
    .slice(0, 2000)

  const occurredAt = parseInt(thread.messages[0].internalDate, 10)
  return { content, occurredAt, replyCount: thread.messages.length }
}

// src/services/google/drive.ts (~80 lines)

export async function listNewBrainFiles(
  folderId: string,
  accessToken: string,
  sinceMs: number
): Promise<GoogleDriveFile[]> {
  const modifiedAfter = new Date(sinceMs).toISOString()
  const res = await fetch(
    `https://www.googleapis.com/drive/v3/files?` +
    `q='${folderId}'+in+parents+and+mimeType='text/markdown'` +
    `+and+modifiedTime>'${modifiedAfter}'` +
    `&fields=files(id,name,modifiedTime)`,
    { headers: { Authorization: `Bearer ${accessToken}` } }
  )
  if (!res.ok) throw new Error(`Drive API error: ${res.status}`)
  const data = await res.json() as { files: GoogleDriveFile[] }
  return data.files
}

export async function downloadDriveFile(
  fileId: string,
  accessToken: string
): Promise<string> {
  const res = await fetch(
    `https://www.googleapis.com/drive/v3/files/${fileId}?alt=media`,
    { headers: { Authorization: `Bearer ${accessToken}` } }
  )
  if (!res.ok) throw new Error(`Drive download error: ${res.status}`)
  return res.text()
}

export function parseObsidianFrontmatter(
  content: string
): { brainFlag: boolean; generatedByBrain: boolean; metadata: Record<string, string> } {
  const fmMatch = content.match(/^---\n([\s\S]*?)\n---/)
  if (!fmMatch) return { brainFlag: false, generatedByBrain: false, metadata: {} }

  const lines = fmMatch[1].split('\n')
  const metadata: Record<string, string> = {}
  for (const line of lines) {
    const [key, ...rest] = line.split(':')
    if (key && rest.length) metadata[key.trim()] = rest.join(':').trim()
  }

  return {
    brainFlag: metadata['brain'] === 'true',
    generatedByBrain: metadata['generated_by'] === 'the-brain',  // Anti-circular
    metadata,
  }
}
```
