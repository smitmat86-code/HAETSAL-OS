## What We're Building

The first domain agent. Career Coach is a `BaseAgent` subclass scoped to the
`'career'` domain. It has a career-specific system prompt, loads career domain
memories at `open()`, and writes career session synthesis at `close()`. It can
propose calendar events (meeting scheduling) and messages (follow-ups) via the
action tool stubs from 1.3.

Two things make Career Coach more than just a system prompt:

1. **Domain mental model reads at `open()`** — Career Coach loads not just recent
   episodic memories but the tenant's career mental model (goals, key relationships,
   active projects) as semantic context. This makes every session aware of the
   bigger picture, not just recent events.

2. **Session synthesis at `close()`** — after every Career Coach session, a
   structured episodic memory is written: what was discussed, what decisions were
   made, what actions were proposed. This feeds the nightly cron's behavioral
   pattern extraction (Pass 4 in 3.3).

When complete: saying "help me prep for my performance review" routes to Career
Coach, which loads relevant career memories, helps the tenant think through it,
proposes a calendar block to prepare, and writes a session synthesis that the
nightly cron will incorporate into the tenant's career mental model.

## THE Brain Laws Check

### Law 1: One Public Face
No new HTTP surface. Career Coach is instantiated within McpAgentDO — same
session path as Chief of Staff.

### Law 2: Zero-Knowledge
Career Coach reads from Hindsight (encrypted recall → decrypt in DO).
Career Coach writes to Hindsight via `BaseAgent.retain()` → `retainContent()`
(encrypt in DO before retain). Reasoning trace encrypted at `close()`.
No career content ever in D1 or plaintext outside DO memory.

### Law 3: Agent Write Policy (Structural)
Inherited from `BaseAgent`. Career Coach can only call `this.retain()` which
accepts `EpistemicMemoryType` — `'episodic' | 'semantic' | 'world'` only.
Career domain synthesis is `'episodic'`. Career goal updates are `'semantic'`.
No procedural writes possible.

## Career Coach Design

### System Prompt

```typescript
protected systemPrompt(): string {
  return `You are THE Brain's Career Coach — a trusted advisor on professional
growth, work challenges, and career strategy.

You know ${this.tenantName}'s career context deeply:
${this.contextSummary()}

Active career goals: ${this.careerGoals()}
Key professional relationships: ${this.keyRelationships()}
Pending career actions: ${this.context.pendingActions.filter(a => a.integration === 'calendar' || a.integration === 'email').length} awaiting approval

Your approach:
- Listen first. Understand what's actually being asked before advising.
- Connect present situations to longer-term goals.
- Propose concrete next steps — don't just discuss abstractly.
- When a concrete action makes sense, propose it via brain_v1_act_* tools.
- Surface relevant past decisions or patterns from memory when they're useful.

You write session memories about: decisions made, insights reached, actions proposed.
You do NOT write personality rules or behavioral patterns — the nightly process does that.

Domain: career | Trace: ${this.traceId}`
}
```

### `open()` — Career-Specific Memory Load

Career Coach overrides `open()` to load career-specific context beyond the
BaseAgent defaults:

```typescript
async open(): Promise<void> {
  await super.open()  // BaseAgent loads generic context

  // Career-specific: load key relationships (semantic memories tagged 'relationships')
  const relationshipResults = await this.hindsightRecall({
    tenant_id: this.hindsightTenantId,
    query_encrypted: await this.encryptQuery('professional relationships colleagues managers clients'),
    domain: 'career',
    limit: 8,
  })

  // Career-specific: load recent decisions (episodic)
  const decisionResults = await this.hindsightRecall({
    tenant_id: this.hindsightTenantId,
    query_encrypted: await this.encryptQuery('career decisions commitments agreed'),
    domain: 'career',
    limit: 5,
  })

  this.context.careerRelationships = this.decryptResults(relationshipResults)
  this.context.recentDecisions = this.decryptResults(decisionResults)
}
```

### `close()` — Career Session Synthesis

```typescript
async close(): Promise<void> {
  // Structured synthesis — feeds nightly cron Pass 4 (behavioral patterns)
  const synthesis = this.buildCareerSynthesis()

  await this.retain(
    synthesis,
    'episodic',
    'career',
    'career_coach_session'
  )

  await super.close(synthesis)  // BaseAgent handles trace + state clear
}

private buildCareerSynthesis(): string {
  const actions = this.sessionProposedActions
  const topics = this.sessionTopics

  return [
    `Career session — ${new Date().toISOString()}`,
    topics.length ? `Topics: ${topics.join(', ')}` : '',
    actions.length ? `Proposed actions: ${actions.map(a => a.tool_name).join(', ')}` : '',
    this.sessionKeyInsight ? `Key insight: ${this.sessionKeyInsight}` : '',
  ].filter(Boolean).join('\n')
}
```

### Tool Registration

Career Coach registers two additional MCP tools beyond the BaseAgent defaults
(`memory_search`, `memory_write`):

```typescript
// In McpAgent.ts — registered when Career Coach session is active
// These are session-scoped tools, not always-available

this.server.tool(
  'career_context',
  'Retrieve current career goals, active projects, and key professional relationships',
  z.object({}).shape,
  async () => {
    return { content: this.careerAgent.getCareerContext() }
  }
)

this.server.tool(
  'career_note',
  'Write a career insight or observation to memory',
  z.object({
    content: z.string(),
    memory_type: z.enum(['episodic', 'semantic']),
    note: z.string().optional(),
  }).shape,
  async ({ content, memory_type, note }) => {
    const result = await this.careerAgent.retain(
      content,
      memory_type as EpistemicMemoryType,
      'career',
      'career_coach_tool'
    )
    return { memory_id: result?.memoryId ?? null }
  }
)
```

## Memory Interface (Formalized in this session)

The 2-tool memory interface exposed to MCP callers (finalized here — referenced
in build sequence as "2-tool memory interface"):

| Tool | Description | Inputs |
|------|-------------|--------|
| `memory_search` | Search memories by query | `query: string, domain?: string, limit?: number` |
| `memory_write` | Write an episodic or semantic memory | `content: string, memory_type: 'episodic'\|'semantic', domain?: string` |

`memory_search` calls Hindsight recall, decrypts results, returns plaintext to
MCP caller. `memory_write` calls `retainContent()` via agent context.

These are registered on McpAgentDO for ALL sessions (not just Career Coach) —
they're the tenant's direct memory interface. Career Coach and other domain agents
use `BaseAgent.retain()` internally, separate from these MCP-facing tools.
