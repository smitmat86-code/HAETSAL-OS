## What We're Building

The first and only public surface of THE Brain: the McpAgent Durable Object,
exposed at `/mcp` via Streamable HTTP transport. This session wires CF Access JWT
validation, derives the Tenant Master Key (TMK) from the verified identity, holds
the TMK in DO memory only (never written to disk), provisions the Cron KEK on first
auth, and registers memory tool stubs (`brain_v1_retain`, `brain_v1_recall`) that
return structured responses but do not yet call Hindsight. It also establishes the
AI Gateway (`brain-gateway`) as the mandatory route for all LLM calls, and wires
a WebSocket upgrade handler in the DO for Pages UI push (used in 1.3+).

When complete: a Claude client (Claude.ai or Claude Desktop) can connect to `/mcp`,
authenticate via CF Access, and invoke `brain_v1_retain` / `brain_v1_recall` with
stub responses. The connection is authenticated, audited, and rate-limited. Nothing
is written to Hindsight yet — that wires in Phase 2.

## THE Brain Laws Check

### Law 1: One Public Face
This spec creates THE Brain's only public surface: the McpAgent Worker at `/mcp`.
Everything behind it is internal:
- Hindsight Container: service binding only (confirmed private by default in 1.1)
- Neon via Hyperdrive: internal to Container
- D1, KV, R2: Worker bindings, not public
- The McpAgent DO itself: not directly addressable — routed through the Worker

The health check Worker from 1.1 is **deleted** in this session. It has no auth
and must not persist. Replace `main` in `wrangler.toml` with the new McpAgent Worker.

### Law 2: Zero-Knowledge
TMK derivation and storage:
- TMK is derived from the CF Access JWT sub claim (deterministic HKDF-SHA256)
- TMK is held **in DO memory only** — `this.tmk` on the McpAgent DO instance
- TMK is **never written** to D1, KV, R2, or any persistent store
- TMK is re-derived on every cold DO start (DO eviction = automatic key rotation)

Cron KEK provisioning:
- On first successful auth, if `tenants.cron_kek_encrypted` is NULL: generate a
  random 256-bit KEK, encrypt it with the TMK (AES-256-GCM), store the ciphertext
  in `tenants.cron_kek_encrypted`
- On subsequent auths, `cron_kek_expires_at` is checked — if within 2h of expiry,
  re-encrypt with fresh IV and extend by 24h
- The plaintext KEK never leaves the DO memory context where it was decrypted

D1 audit tables: metadata only — no content fields. Confirmed by 1.1 postflight.

### Law 3: Agent Write Policy
No agents write memory in this session. Tool stubs return mock responses.
The write policy validator (heuristic + classifier) is **not yet wired** — it
belongs in Phase 2.1 when real Hindsight writes begin. Stub `brain_v1_retain`
must include a `// TODO: Phase 2.1 — wire write policy validator before real retain`
comment so the gap is discoverable.

## Behavioral Wiring

### Authorization

| Route / Entry Point | Auth | Tenant ID Source | Notes |
|---------------------|------|-----------------|-------|
| `POST /mcp` | CF Access JWT (mandatory) | JWT `sub` claim → HKDF | Streamable HTTP transport |
| `GET /mcp` | CF Access JWT (mandatory) | JWT `sub` claim → HKDF | SSE transport (future) |
| `GET /ws` | CF Access JWT (mandatory) | JWT `sub` claim → HKDF | WebSocket upgrade for Pages |
| `GET /health` | **Removed** | — | Health Worker deleted in this session |

**JWT validation:** Use the `CF-Access-Jwt-Assertion` header. Validate signature
against Cloudflare's JWKs endpoint for your Access application audience. Extract
`sub` as the stable tenant identity. Never trust `tenant_id` from any other source.

**Tenant bootstrap:** On first auth (tenant row not yet in D1), create the tenant
row atomically with the Hindsight tenant registration call. If either fails, roll
back both (D1 batch + Hindsight registration in a compensating transaction pattern).

### Action Layer
No action tools in this spec.

### Audit

| Event | Table | Action String | Blocking? |
|-------|-------|--------------|-----------|
| Auth success (first ever) | `memory_audit` | `auth.tenant_created` | Atomic with tenant INSERT |
| Auth success | `agent_traces` | — | `waitUntil()` — non-blocking |
| `brain_v1_retain` invoked (stub) | `memory_audit` | `memory.retain_stub` | `waitUntil()` |
| `brain_v1_recall` invoked (stub) | `memory_audit` | `memory.recall_stub` | `waitUntil()` |
| KEK provisioned | `memory_audit` | `auth.kek_provisioned` | Atomic with tenants UPDATE |
| KEK renewed | `memory_audit` | `auth.kek_renewed` | Atomic with tenants UPDATE |
| JWT validation failure | `memory_audit` | `auth.jwt_invalid` | `waitUntil()` |

All `waitUntil()` writes use `c.executionCtx.waitUntil()` — never block MCP
response on audit writes.

### Compute Routing

| Operation | Tier | Rationale |
|-----------|------|-----------|
| JWT validation | Sync (C1) | <1ms, in-memory |
| TMK derivation (HKDF) | Sync (C1) | Crypto.subtle, <1ms |
| Tenant bootstrap (first auth) | Sync (C1) | Blocking — must complete before responding |
| KEK provision/renewal | Sync (C1) | Blocking — must complete before tool calls |
| Tool stub responses | Sync (C1) | No Hindsight call yet |
| Audit writes | `waitUntil()` | Non-blocking, best-effort |

### Other Behavioral Wiring

**Agent identity for stubs:** `agent_identity = 'mcpagent/stub'` in audit records
until real agents replace stubs in Phase 2+.

**AI Gateway:** All LLM calls in this and all future sessions MUST use:
```typescript
env.AI.run(model, input, { gateway: { id: 'brain-gateway' } })
```
No direct Anthropic/OpenAI API calls. AI Gateway is created manually in Cloudflare
dashboard before first deploy (name: `brain-gateway`).

**Platform default scheduled_tasks:** On tenant creation (first auth), seed these
rows into `scheduled_tasks`:
```
consolidation_cron     — '0 3 * * *'    (3 AM UTC daily)
morning_brief          — '0 7 * * *'    (7 AM UTC daily)
gap_discovery          — '0 4 * * 0'    (4 AM UTC Sunday)
weekly_synthesis       — '0 8 * * 1'    (8 AM UTC Monday)

All `is_platform_default = 1`, `enabled = 1`. Seeded in same D1 batch as tenant INSERT.

**WebSocket push stub:** The DO handles WebSocket upgrade at `/ws`. In this session:
accept the connection, hold it in `this.wsConnections: Set<WebSocket>`, send a
`{ type: 'connected', tenantId }` message on open. No other push logic until 1.3.

// src/middleware/auth.ts (~80 lines)
// CF Access JWT validation + TMK derivation
// LESSON: TenantContext Wiring — stamp onto Hono context here.
//         Routes without auditMiddleware must access c.get('tenantContext') directly.

import { Context, MiddlewareHandler, Next } from 'hono'
import type { Env } from '../types/env'
import type { TenantContext } from '../types/tenant'

// HKDF-SHA256: JWT sub → tenant_id
// Deterministic — same sub always produces same tenant_id
// DO NOT store tenantId in D1 as the JWT sub directly (PII risk)
async function deriveTeantId(sub: string, secret: string): Promise<string> {
  const keyMaterial = await crypto.subtle.importKey(
    'raw',
    new TextEncoder().encode(secret),
    { name: 'HKDF' },
    false,
    ['deriveBits']
  )
  const bits = await crypto.subtle.deriveBits(
    { name: 'HKDF', hash: 'SHA-256', salt: new TextEncoder().encode('brain-tenant-id'), info: new TextEncoder().encode(sub) },
    keyMaterial,
    256
  )
  return Array.from(new Uint8Array(bits)).map(b => b.toString(16).padStart(2, '0')).join('')
}

// TMK derivation — held in DO memory only, never written to disk
// Called from McpAgent DO on each new session
export async function deriveTmk(sub: string, secret: string): Promise<CryptoKey> {
  const keyMaterial = await crypto.subtle.importKey(
    'raw', new TextEncoder().encode(secret), { name: 'HKDF' }, false, ['deriveKey']
  )
  return crypto.subtle.deriveKey(
    { name: 'HKDF', hash: 'SHA-256', salt: new TextEncoder().encode('brain-tmk'), info: new TextEncoder().encode(sub) },
    keyMaterial,
    { name: 'AES-GCM', length: 256 },
    false,  // NOT extractable — can never be serialized out of memory
    ['encrypt', 'decrypt']
  )
}

export function authMiddleware(): MiddlewareHandler<{ Bindings: Env }> {
  return async (c, next) => {
    const jwt = c.req.header('CF-Access-Jwt-Assertion')
    if (!jwt) return c.json({ error: 'Unauthorized' }, 401)

    // Validate JWT against Cloudflare JWKs
    // JWKs URL: https://<CF_ACCESS_TEAM>.cloudflareaccess.com/cdn-cgi/access/certs
    const jwksUrl = `https://${c.env.CF_ACCESS_TEAM}.cloudflareaccess.com/cdn-cgi/access/certs`
    let payload: { sub: string; aud: string | string[]; exp: number }

    try {
      // Use Web Crypto to validate — no external JWT library needed
      payload = await validateCfAccessJwt(jwt, jwksUrl, c.env.CF_ACCESS_AUD)
    } catch {
      // LESSON: waitUntil for audit — don't block rejection on audit write
      c.executionCtx.waitUntil(writeAuditLog(c.env, 'auth.jwt_invalid', null))
      return c.json({ error: 'Unauthorized' }, 401)
    }

    const tenantId = await deriveTenantId(payload.sub, c.env.CF_ACCESS_AUD)
    // TenantContext stamped here — available to all downstream middleware and routes
    c.set('tenantId', tenantId)
    c.set('jwtSub', payload.sub)

    await next()
  }
}
```

// src/services/tenant.ts (~100 lines)
// Tenant bootstrap, KEK provision/renewal, scheduled_tasks seed
// All D1 writes are batched — never sequential for paired operations

import type { Env } from '../types/env'
import type { TenantRow, TenantContext } from '../types/tenant'

const PLATFORM_DEFAULT_TASKS = [
  { task_name: 'consolidation_cron', cron_expression: '0 3 * * *',  description: 'Nightly memory consolidation' },
  { task_name: 'morning_brief',      cron_expression: '0 7 * * *',  description: 'Daily morning brief generation' },
  { task_name: 'gap_discovery',      cron_expression: '0 4 * * 0',  description: 'Weekly knowledge gap discovery' },
  { task_name: 'weekly_synthesis',   cron_expression: '0 8 * * 1',  description: 'Weekly synthesis and reflection' },
]

export async function getOrCreateTenant(
  tenantId: string,
  jwtSub: string,
  env: Env
): Promise<{ tenant: TenantRow; isNew: boolean }> {
  const db = env.D1_US  // TODO: Phase 5+ — route by data_region for DLS
  const existing = await db.prepare(
    'SELECT * FROM tenants WHERE id = ?'
  ).bind(tenantId).first<TenantRow>()

  if (existing) return { tenant: existing, isNew: false }

  // First auth — register with Hindsight and create tenant row atomically
  const hindsightTenantId = await registerHindsightTenant(tenantId, env)

  const now = Date.now()
  const taskRows = PLATFORM_DEFAULT_TASKS.map(t => ({
    id: crypto.randomUUID(),
    tenant_id: tenantId,
    task_name: t.task_name,
    cron_expression: t.cron_expression,
    enabled: 1,
    is_platform_default: 1,
    description: t.description,
    scope_json: null,
    last_run_at: null,
    next_run_at: null,
    created_at: now,
    updated_at: now,
  }))

  // Atomic: tenant INSERT + all scheduled_tasks + audit record in one batch
  // LESSON: Atomic — no TOCTOU, no paired sequential writes (Schema V2 Spec 1.5)
  await db.batch([
    db.prepare(
      `INSERT INTO tenants (id, created_at, updated_at, hindsight_tenant_id, ai_cost_reset_at)
       VALUES (?, ?, ?, ?, ?)`
    ).bind(tenantId, now, now, hindsightTenantId, now),
    ...taskRows.map(t =>
      db.prepare(
        `INSERT INTO scheduled_tasks
         (id, tenant_id, task_name, cron_expression, enabled, is_platform_default,
          description, created_at, updated_at)
         VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?)`
      ).bind(t.id, t.tenant_id, t.task_name, t.cron_expression, t.enabled,
             t.is_platform_default, t.description, t.created_at, t.updated_at)
    ),
    db.prepare(
      `INSERT INTO memory_audit (id, tenant_id, created_at, operation)
       VALUES (?, ?, ?, ?)`
    ).bind(crypto.randomUUID(), tenantId, now, 'auth.tenant_created'),
  ])

  return {
    tenant: await db.prepare('SELECT * FROM tenants WHERE id = ?').bind(tenantId).first<TenantRow>() as TenantRow,
    isNew: true,
  }
}

export async function provisionOrRenewKek(
  tenant: TenantRow,
  tmk: CryptoKey,
  env: Env
): Promise<void> {
  const db = env.D1_US
  const now = Date.now()
  const TWO_HOURS_MS = 2 * 60 * 60 * 1000
  const TWENTY_FOUR_HOURS_MS = 24 * 60 * 60 * 1000

  const needsProvision = !tenant.cron_kek_encrypted
  const needsRenewal = tenant.cron_kek_expires_at
    ? tenant.cron_kek_expires_at - now < TWO_HOURS_MS
    : false

  if (!needsProvision && !needsRenewal) return

  // Generate or re-encrypt KEK
  const kekBytes = needsProvision
    ? crypto.getRandomValues(new Uint8Array(32))
    : await decryptKek(tenant.cron_kek_encrypted!, tmk)  // re-encrypt with fresh IV

  const iv = crypto.getRandomValues(new Uint8Array(12))
  const encrypted = await crypto.subtle.encrypt(
    { name: 'AES-GCM', iv },
    tmk,
    kekBytes
  )
  const ciphertext = btoa(String.fromCharCode(...new Uint8Array([...iv, ...new Uint8Array(encrypted)])))

  const operation = needsProvision ? 'auth.kek_provisioned' : 'auth.kek_renewed'

  // Atomic: tenants UPDATE + audit in one batch
  await db.batch([
    db.prepare(
      `UPDATE tenants
       SET cron_kek_encrypted = ?, cron_kek_expires_at = ?, updated_at = ?
       WHERE id = ?`
    ).bind(ciphertext, now + TWENTY_FOUR_HOURS_MS, now, tenant.id),
    db.prepare(
      `INSERT INTO memory_audit (id, tenant_id, created_at, operation)
       VALUES (?, ?, ?, ?)`
    ).bind(crypto.randomUUID(), tenant.id, now, operation),
  ])
}

## Part 5: McpAgent Durable Object

```typescript
// src/workers/mcpagent/do/McpAgent.ts (~150 lines)
// THE Brain's session container
// Holds TMK in memory — eviction = key rotation
// Handles MCP Streamable HTTP + WebSocket upgrade for Pages push

import { McpAgent as BaseMcpAgent } from 'agents/mcp'
import type { Env } from '../../../types/env'
import { deriveTmk } from '../../../middleware/auth'
import { getOrCreateTenant, provisionOrRenewKek } from '../../../services/tenant'
import { retainStub } from '../../../tools/retain'
import { recallStub } from '../../../tools/recall'

export class McpAgent extends BaseMcpAgent<Env> {
  // TMK held in DO memory only — NEVER written to storage
  // Eviction clears it — re-derived on next auth
  private tmk: CryptoKey | null = null
  private tenantId: string | null = null
  private wsConnections: Set<WebSocket> = new Set()

  async init(jwtSub: string, tenantId: string, env: Env) {
    // Called once per DO session — derive TMK from JWT sub
    this.tenantId = tenantId
    this.tmk = await deriveTmk(jwtSub, env.CF_ACCESS_AUD)

    // Bootstrap tenant if new; provision/renew KEK
    const { tenant } = await getOrCreateTenant(tenantId, jwtSub, env)
    await provisionOrRenewKek(tenant, this.tmk, env)

    // Register MCP tools
    this.server.tool('brain_v1_retain', 'Retain a memory in THE Brain', retainSchema, retainStub)
    this.server.tool('brain_v1_recall', 'Recall memories from THE Brain', recallSchema, recallStub)
  }

  // WebSocket upgrade handler — Pages UI push (1.3+ will broadcast action events)
  async fetch(request: Request, env: Env): Promise<Response> {
    const url = new URL(request.url)

    // LESSON: WebSocket 101 headers are immutable in workerd
    // Security-header middleware MUST skip mutation on upgrade responses
    if (url.pathname === '/ws' && request.headers.get('Upgrade') === 'websocket') {
      const [client, server] = Object.values(new WebSocketPair())
      server.accept()
      this.wsConnections.add(server)

      server.addEventListener('message', (event) => {
        // Ping/pong only in Phase 1 — action events wired in 1.3
      })
      server.addEventListener('close', () => {
        this.wsConnections.delete(server)
      })

      // Send connected confirmation
      server.send(JSON.stringify({ type: 'connected', tenantId: this.tenantId }))

      // LESSON: Use new Request(url, request) when forwarding to DO to preserve
      // upgrade semantics — do not reconstruct from individual headers
      return new Response(null, { status: 101, webSocket: client })
    }

    // Delegate MCP Streamable HTTP to base class
    return super.fetch(request, env)
  }

  // Broadcast to all connected Pages clients — called by Action Worker in 1.3
  broadcast(message: unknown) {
    const payload = JSON.stringify(message)
    for (const ws of this.wsConnections) {
      try { ws.send(payload) } catch { this.wsConnections.delete(ws) }
    }
  }
}
```

// Hono app — route registrations only
// Middleware order: auth → audit → dlp (MCP routes) → handler
// LESSON: Security headers in try/finally — skip on WebSocket 101 responses

import { Hono } from 'hono'
import { McpAgent } from './do/McpAgent'
import { authMiddleware } from '../../middleware/auth'
import { auditMiddleware } from '../../middleware/audit'
import { dlpMiddleware } from '../../middleware/dlp'
import type { Env } from '../../types/env'

const app = new Hono<{ Bindings: Env }>()

// Security headers — try/finally so error responses are also hardened
// LESSON: Skip header mutation for WebSocket 101 responses (immutable in workerd)
app.use('*', async (c, next) => {
  const isWebSocket = c.req.header('Upgrade') === 'websocket'
  await next()
  if (!isWebSocket) {
    c.header('X-Content-Type-Options', 'nosniff')
    c.header('X-Frame-Options', 'DENY')
    c.header('Referrer-Policy', 'no-referrer')
  }
})

// Auth on all routes
app.use('*', authMiddleware())

// Audit on all routes (stamps tenantContext after auth)
app.use('*', auditMiddleware())

// DLP only on MCP routes
app.use('/mcp/*', dlpMiddleware())

// MCP Streamable HTTP — delegate to DO
app.all('/mcp/*', async (c) => {
  const id = c.env.MCPAGENT.idFromName(c.get('tenantId'))
  const stub = c.env.MCPAGENT.get(id)
  return stub.fetch(c.req.raw)
})

// WebSocket upgrade — delegate to DO
app.get('/ws', async (c) => {
  const id = c.env.MCPAGENT.idFromName(c.get('tenantId'))
  const stub = c.env.MCPAGENT.get(id)
  // LESSON: Use new Request(url, c.req.raw) to preserve WebSocket upgrade semantics
  return stub.fetch(new Request(c.req.url, c.req.raw))
})

export { McpAgent }
export default app

- **`cloudflare:test` ProvidedEnv** needs explicit type augmentation. Created `tests/env.d.ts` to extend ProvidedEnv with project Env.
- **`zod@3.25.1`** installed as peer dep for agents SDK — tool schemas require Zod.
- **DO RPC methods** (initTenant, handleWebSocket, broadcast) require `@ts-expect-error` when called via generic `DurableObjectStub` — workerd types don't know about custom RPC methods.
