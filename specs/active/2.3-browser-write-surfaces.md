## What We're Building

The first actions that actually do something: a real browser, and real calendar
writes. `brain_v1_act_browse` uses Cloudflare Browser Rendering (Puppeteer API
via BROWSER binding) to navigate to a URL, extract page content, and return it to
the calling agent. `brain_v1_act_create_event` and `brain_v1_act_modify_event`
use the Google Calendar API (tokens from 2.2) to create and modify events.

Both write surfaces include: automatic episodic memory on successful execution
(the `// TODO: Phase 2.1` comment finally resolves), a 5-minute undo window for
reversible actions, and WebSocket push to the Pages UI on state changes.

When complete: an agent can browse a URL and get structured content back. An agent
can propose creating a calendar event — it routes YELLOW (WRITE_EXTERNAL_REVERSIBLE),
waits for Pages UI approval (1.4 closes the loop), and on approval executes the
real Google Calendar API call.

## THE Brain Laws Check

### Law 1: One Public Face
**CRITICAL:** BROWSER binding is on the McpAgent Worker — NOT in the Hindsight
Container. The Container has no BROWSER binding and cannot call Browser Rendering.
The architecture note from the build sequence explicitly flags this.

The browse action flow:
```
McpAgent DO (tool call) → QUEUE_ACTIONS → Action Worker →
Action Worker calls BROWSER binding directly (Worker-side) →
Returns extracted content to McpAgentDO via DO RPC result
```

No new public HTTP surface in this session.

### Law 2: Zero-Knowledge
Browse results (page content) are:
1. Returned to the calling agent (in-session, not persisted unless agent retains)
2. If the agent subsequently calls `brain_v1_retain` with browse content → normal
   encrypt-before-retain path from 2.1
3. Screenshots stored in R2_ARTIFACTS encrypted with TMK if capture requested

Calendar event writes: event content (title, description) encrypted before storage
in `pending_actions` payload (R2). The Google Calendar API itself receives plaintext
(it must — it's an external service). This is the documented boundary: content
leaves the encrypted domain when it crosses to an external service. The episodic
memory written after execution is encrypted before Hindsight.

### Law 3: Agent Write Policy
Episodic memory written on successful action execution (Law 3 compliant: episodic
only, no procedural). The automatic memory content is templated metadata:
`"Sent/Created/Browsed [action_type] via [integration] at [timestamp]"` —
not the actual content of the action. Content is in the action payload (R2).

**5-minute undo window:**
After execution of WRITE_EXTERNAL_REVERSIBLE actions:
1. Action state transitions to `completed_reversible` (not `completed`)
2. WebSocket push includes `undo_expires_at: now + 300_000`
3. Pages UI (1.4) shows undo button until expiry
4. `POST /actions/{id}/undo` route (new in this session) calls the reversal API
5. After 5 minutes: background cron transitions `completed_reversible` → `completed`
   (TODO: Phase 1.4 cron or explicit state machine)

New `pending_actions.state` value: `completed_reversible` — add to the state enum
in `src/types/action.ts`. Must not break existing state machine logic.

### Other Behavioral Wiring

**Browse execution pattern:**
```typescript
// src/services/action/integrations/browser.ts
// LESSON: BROWSER binding is on Worker, not Container
// Action Worker has BROWSER binding — confirmed from wrangler.toml 1.1

import puppeteer from '@cloudflare/puppeteer'

export async function executeBrowse(
  url: string,
  env: Env
): Promise<{ content: string; title: string; screenshot?: string }> {
  const browser = await puppeteer.launch(env.BROWSER)
  try {
    const page = await browser.newPage()
    await page.goto(url, { waitUntil: 'networkidle0', timeout: 30_000 })
    const title = await page.title()
    // Extract readable content — strip scripts, styles, nav
    const content = await page.evaluate(() => {
      const body = document.querySelector('main') ?? document.body
      return body?.innerText?.slice(0, 3000) ?? ''
    })
    return { content, title }
  } finally {
    await browser.close()
  }
}
```

**Calendar create/modify pattern:**
```typescript
// src/services/action/integrations/calendar.ts

export async function executeCreateEvent(
  input: z.infer<typeof createEventSchema>,
  tenantId: string,
  env: Env
): Promise<{ eventId: string; htmlLink: string }> {
  const accessToken = await getGoogleToken(tenantId, 'calendar', env)

  const res = await fetch(
    'https://www.googleapis.com/calendar/v3/calendars/primary/events',
    {
      method: 'POST',
      headers: {
        Authorization: `Bearer ${accessToken}`,
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        summary: input.title,
        description: input.description,
        start: { dateTime: input.start_time, timeZone: input.timezone ?? 'UTC' },
        end: { dateTime: input.end_time, timeZone: input.timezone ?? 'UTC' },
        attendees: input.attendees?.map(email => ({ email })),
      })
    }
  )
  if (!res.ok) throw new Error(`Calendar API error: ${res.status}`)
  const event = await res.json() as { id: string; htmlLink: string }
  return { eventId: event.id, htmlLink: event.htmlLink }
}
```

**Automatic episodic memory on execution:**
```typescript
// Replaces // TODO: Phase 2.1 comment in executor.ts
// Called from executeStub() → now executeReal() for these three tools

async function writeActionEpisodicMemory(
  msg: ActionQueueMessage,
  result: { eventId?: string; htmlLink?: string },
  tmk: CryptoKey,
  env: Env
) {
  const artifact: IngestionArtifact = {
    source: 'mcp_retain',
    tenantId: msg.tenant_id,
    hindsightTenantId: await getHindsightTenantId(msg.tenant_id, env),
    rawContent: `Action executed: ${msg.tool_name} via ${msg.integration} at ${new Date().toISOString()}`,
    occurredAt: Date.now(),
    provenance: 'action_execution',
    memoryType: 'episodic',
    domain: 'general',
    metadata: {
      action_id: msg.action_id,
      tool_name: msg.tool_name,
      integration: msg.integration,
      ...(result.htmlLink ? { external_link: result.htmlLink } : {}),
    }
  }
  // Use retainContent from 2.1 — same encrypted path
  const retained = await retainContent(artifact, tmk, env)
  if (retained) {
    // Update pending_actions.episodic_memory_id
    await env.D1_US.prepare(
      'UPDATE pending_actions SET episodic_memory_id = ? WHERE id = ?'
    ).bind(retained.memoryId, msg.action_id).run()
  }
}
```
