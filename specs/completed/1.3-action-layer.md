## What We're Building

The authorization gate that stands between an agent's intention and the real world.
The Action Worker reads from `QUEUE_ACTIONS` only — it has no HTTP surface. When an
agent proposes an action (via `brain_v1_act_*` tool stubs registered in 1.2), the
proposal lands in the queue. The Action Worker picks it up, looks up the tenant's
authorization preferences (HMAC-verified), classifies the capability class, enforces
the hard floor, and either executes immediately (GREEN), queues for approval (YELLOW),
or blocks (RED). TOCTOU protection hashes the payload at proposal time and verifies
at execution. WebSocket push notifies the Pages UI of state changes (used in 1.4).

When complete: an agent can propose an action, the authorization gate routes it
correctly, YELLOW actions appear in the `pending_actions` table awaiting approval,
GREEN actions execute (stubbed — real integrations wire in Phase 2), and the Pages
UI receives a WebSocket push on every state change. RED actions are logged and
dropped. The full audit trail is written atomically with every state transition.

## THE Brain Laws Check

### Law 1: One Public Face
The Action Worker has **no HTTP surface**. It is a queue consumer only — not
accessible via any URL. It communicates outward only via:
- DO RPC to `McpAgentDO.broadcast()` for WebSocket push
- D1 writes for state and audit
- External integration calls (stubbed — Phase 2)

The McpAgent Worker (1.2) remains the only public surface. `brain_v1_act_*` tool
stubs are registered on the McpAgent DO. The McpAgent publishes to `QUEUE_ACTIONS`;
the Action Worker consumes. The queue is the boundary — no Worker-to-Worker HTTP.

### Law 2: Zero-Knowledge
Action payloads (email body, message text, calendar event content) are encrypted
with the tenant TMK before being written to R2. `pending_actions.payload_r2_key`
holds the R2 key; `pending_actions.payload_hash` holds SHA-256 of the **plaintext**
payload (for TOCTOU). D1 holds metadata and state only — never payload content.

The TMK is not available to the Action Worker directly (it lives in the McpAgent DO
memory). Payload encryption happens in the McpAgent DO **before** publishing to the
queue. The Action Worker receives encrypted payloads in R2 and a hash — it never
sees plaintext.

In this session, `brain_v1_act_*` tools are stubs with no real payload — a dummy
`payload` string is used to exercise the TOCTOU and authorization flows. Real
encryption wires in Phase 2 when actual integrations exist.

### Law 3: Agent Write Policy
On successful action execution, the Action Worker writes an episodic memory to
Hindsight (stub in this session — `// TODO: Phase 2.1`). This write is episodic
only — procedural memory is never written by the Action Worker. The write policy
validator is not in this path (actions are not memory retain operations).

## Behavioral Wiring

### Authorization

The Action Worker has no routes. Authorization is enforced inside the queue
consumer handler, not via HTTP middleware.

| Entry Point | Auth | Tenant ID Source | When Checked |
|-------------|------|-----------------|-------------|
| Queue consumer | HMAC verification of `tenant_action_preferences` row | `pending_actions.tenant_id` (from queue message) | Before every execution |
| DO RPC `broadcast()` | Implicit — only Action Worker calls it; Worker has MCPAGENT binding | `tenantId` from queue message | At broadcast call |

**Hard floors (non-negotiable):**

| Capability Class | Hard Floor | Cannot Be Lowered To |
|-----------------|-----------|---------------------|
| `READ` | GREEN | — |
| `WRITE_INTERNAL` | GREEN | — |
| `WRITE_EXTERNAL_REVERSIBLE` | YELLOW | GREEN |
| `WRITE_EXTERNAL_IRREVERSIBLE` | YELLOW | GREEN |
| `WRITE_EXTERNAL_FINANCIAL` | RED | YELLOW or GREEN |
| `DELETE` | RED | YELLOW or GREEN |

The authorization gate derives the effective level as:
`max(tenant_preference_level, hard_floor)` where RED > YELLOW > GREEN.
A tenant can only raise their floor, never lower it below the hard floor.
HMAC failure on `tenant_action_preferences` row → treat as RED regardless.

### Audit

| Event | Table | Action String | Blocking? |
|-------|-------|--------------|-----------|

### Other Behavioral Wiring

**Queue message schema (published by McpAgent DO tool stubs):**
```typescript
interface ActionQueueMessage {
  action_id: string          // ulid — pre-generated by tool stub
  tenant_id: string          // from McpAgent context
  proposed_by: string        // agent_identity string (e.g. 'mcpagent/tool')
  tool_name: string          // e.g. 'brain_v1_act_send_message'
  capability_class: string   // pre-classified by tool stub
  integration: string        // 'sms' | 'gmail' | 'calendar' | 'internal' | etc.
  payload_r2_key: string     // R2 key for encrypted payload (stub: placeholder key)
  payload_hash: string       // SHA-256 of plaintext payload (TOCTOU anchor)
  payload_stub: string       // Stub plaintext (Phase 1 only — removed Phase 2)
}
```

**TOCTOU protocol:**
1. Tool stub generates `action_id` (ulid) and hashes dummy payload → `payload_hash`
2. Publishes queue message with `payload_hash`
3. Action Worker INSERTs `pending_actions` row with `payload_hash`
4. At execution time: re-hash `payload_stub` → compare with stored `payload_hash`
   using `crypto.subtle.timingSafeEqual()` — never string equality
5. Mismatch → `action.toctou_violation` anomaly signal + abort

**HMAC protection for `tenant_action_preferences`:**
- Every row has `row_hmac TEXT NOT NULL` (column exists from 1.1 DDL)
- HMAC is `HMAC-SHA256(row_content_canonical_json, HMAC_SECRET)`
- Before reading any preference row: verify HMAC
- Failure → treat effective level as RED (most restrictive), write anomaly signal
- `HMAC_SECRET` comes from `env.HMAC_SECRET` (set in `.dev.vars`, CF secret in prod)

**Default preferences (seeded on first action proposal if no row exists):**
```typescript
// If tenant has no preference row for a capability class, use hard floor as default
// Do NOT seed preference rows at tenant creation — seed on first proposal
// (avoids creating rows the tenant never configured)
```

**WebSocket push events (broadcast via McpAgentDO):**
```typescript
// On YELLOW routing:
{ type: 'action.pending_approval', action_id, tool_name, capability_class, tenant_id }
// On execution (stub):
{ type: 'action.executed', action_id, tool_name, tenant_id }
// On RED block:
{ type: 'action.blocked', action_id, tool_name, capability_class, tenant_id }
// On TOCTOU violation:
{ type: 'action.toctou_violation', action_id, tenant_id }
```

**INSERT OR IGNORE on queue consumer INSERTs:**
All `pending_actions` INSERTs use `INSERT OR IGNORE` — at-least-once queue delivery
means the message may be redelivered. Idempotent INSERT prevents crash on retry.
LESSON applied: Schema V2 Spec 1.5, Bug 3.

**`brain_v1_act_*` tool stubs registration:**
These are registered on `McpAgentDO` in this session (extending 1.2's tool
registration). The tools publish to `QUEUE_ACTIONS` — McpAgent DO needs the
`QUEUE_ACTIONS` binding. Add to DO's env access.

## Part 3: Authorization Service

```typescript
// src/services/action/authorization.ts (~100 lines)
// Auth gate: capability class → effective level
// HMAC verification, hard floor enforcement, preference lookup

import type { Env } from '../../types/env'
import type {
  CapabilityClass, AuthorizationLevel, TenantActionPreferenceRow
} from '../../types/action'
import { AUTH_LEVEL_ORDINAL, HARD_FLOORS, DEFAULT_SEND_DELAY_SECONDS } from '../../types/action'

interface AuthGateResult {
  effectiveLevel: AuthorizationLevel
  sendDelaySeconds: number
  hmacValid: boolean
  preferenceFound: boolean
}

export async function runAuthorizationGate(
  tenantId: string,
  capabilityClass: CapabilityClass,
  integration: string,
  env: Env
): Promise<AuthGateResult> {
  const hardFloor = HARD_FLOORS[capabilityClass]

  // Look up tenant preference — specific integration first, then wildcard
  const pref = await lookupPreference(tenantId, capabilityClass, integration, env)

  if (!pref) {
    // No preference configured — use hard floor as default
    return {
      effectiveLevel: hardFloor,
      sendDelaySeconds: DEFAULT_SEND_DELAY_SECONDS[capabilityClass] ?? 0,
      hmacValid: true,  // No row = no HMAC to verify
      preferenceFound: false,
    }
  }

  // Verify HMAC before trusting the row
  const hmacValid = await verifyPreferenceHmac(pref, env.HMAC_SECRET)

  if (!hmacValid) {
    // HMAC failure — treat as RED regardless of stored value
    return {
      effectiveLevel: 'RED',
      sendDelaySeconds: 0,
      hmacValid: false,
      preferenceFound: true,
    }
  }

  // Effective level = max(preference, hard floor)
  const prefOrdinal = AUTH_LEVEL_ORDINAL[pref.authorization_level]
  const floorOrdinal = AUTH_LEVEL_ORDINAL[hardFloor]
  const effectiveLevel: AuthorizationLevel =
    prefOrdinal >= floorOrdinal ? pref.authorization_level : hardFloor

  return {
    effectiveLevel,
    sendDelaySeconds: pref.send_delay_seconds,
    hmacValid: true,
    preferenceFound: true,
  }
}

async function lookupPreference(
  tenantId: string,
  capabilityClass: CapabilityClass,
  integration: string,
  env: Env
): Promise<TenantActionPreferenceRow | null> {
  // Try specific integration first, fall back to wildcard (integration IS NULL)
  const db = env.D1_US
  const specific = await db.prepare(
    `SELECT * FROM tenant_action_preferences
     WHERE tenant_id = ? AND capability_class = ? AND integration = ?`
  ).bind(tenantId, capabilityClass, integration).first<TenantActionPreferenceRow>()
  if (specific) return specific

  return db.prepare(
    `SELECT * FROM tenant_action_preferences
     WHERE tenant_id = ? AND capability_class = ? AND integration IS NULL`
  ).bind(tenantId, capabilityClass).first<TenantActionPreferenceRow>()
}

async function verifyPreferenceHmac(
  pref: TenantActionPreferenceRow,
  hmacSecret: string
): Promise<boolean> {
  // Canonical JSON — exclude row_hmac field itself
  const canonical = JSON.stringify({
    id: pref.id,
    tenant_id: pref.tenant_id,
    capability_class: pref.capability_class,
    integration: pref.integration,
    authorization_level: pref.authorization_level,
    send_delay_seconds: pref.send_delay_seconds,
    trust_threshold: pref.trust_threshold,
    requires_phrase: pref.requires_phrase,
    created_at: pref.created_at,
  })

  const key = await crypto.subtle.importKey(
    'raw', new TextEncoder().encode(hmacSecret),
    { name: 'HMAC', hash: 'SHA-256' }, false, ['verify']
  )
  const storedHmac = Uint8Array.from(atob(pref.row_hmac), c => c.charCodeAt(0))
  return crypto.subtle.verify('HMAC', key, storedHmac, new TextEncoder().encode(canonical))
}

// Exported for use by Phase 1.4 (settings UI — creating/updating preferences)
export async function computePreferenceHmac(
  pref: Omit<TenantActionPreferenceRow, 'row_hmac'>,
  hmacSecret: string
): Promise<string> {
  const canonical = JSON.stringify({
    id: pref.id, tenant_id: pref.tenant_id,
    capability_class: pref.capability_class,
    integration: pref.integration,
    authorization_level: pref.authorization_level,
    send_delay_seconds: pref.send_delay_seconds,
    trust_threshold: pref.trust_threshold,
    requires_phrase: pref.requires_phrase,
    created_at: pref.created_at,
  })
  const key = await crypto.subtle.importKey(
    'raw', new TextEncoder().encode(hmacSecret),
    { name: 'HMAC', hash: 'SHA-256' }, false, ['sign']
  )
  const sig = await crypto.subtle.sign('HMAC', key, new TextEncoder().encode(canonical))
  return btoa(String.fromCharCode(...new Uint8Array(sig)))
}
```

## Part 4: TOCTOU Service

```typescript
// src/services/action/toctou.ts (~50 lines)

export async function hashPayload(payload: string): Promise<string> {
  const buf = await crypto.subtle.digest('SHA-256', new TextEncoder().encode(payload))
  return Array.from(new Uint8Array(buf)).map(b => b.toString(16).padStart(2, '0')).join('')
}

export async function verifyPayloadHash(
  payload: string,
  storedHash: string
): Promise<boolean> {
  const freshHash = await hashPayload(payload)
  const a = new TextEncoder().encode(freshHash)
  const b = new TextEncoder().encode(storedHash)

  if (a.length !== b.length) return false

  // LESSON: crypto.subtle.timingSafeEqual — never hand-roll constant-time comparison
  return crypto.subtle.timingSafeEqual(a, b)
}
```

// LESSON: INSERT OR IGNORE for all queue consumer INSERTs (at-least-once safety)
// LESSON: Promise.allSettled for fan-out (not sequential for...of)

import type { Env } from '../../types/env'
import type { ActionQueueMessage } from '../../types/action'
import { runAuthorizationGate } from '../../services/action/authorization'
import { verifyPayloadHash } from '../../services/action/toctou'

export default {
  // No fetch handler — this Worker is queue consumer only
  async queue(batch: MessageBatch<ActionQueueMessage>, env: Env): Promise<void> {
    // LESSON: Promise.allSettled for concurrent fan-out
    const results = await Promise.allSettled(
      batch.messages.map(msg => processAction(msg.body, env, msg))
    )
    // Check for failures — don't silently ack partial batch
    const failures = results.filter(r => r.status === 'rejected')
    if (failures.length > 0) {
      // Individual messages are acked/retried via msg.ack()/msg.retry()
      // Batch-level throw only if ALL failed
      if (failures.length === batch.messages.length) {
        throw new Error(`All ${failures.length} action messages failed`)
      }
    }
  }
}

async function processAction(
  msg: ActionQueueMessage,
  env: Env,
  rawMsg: Message<ActionQueueMessage>
): Promise<void> {
  const db = env.D1_US
  const now = Date.now()

  // LESSON: INSERT OR IGNORE — redelivered message is a no-op, not a crash
  const inserted = await db.prepare(
    `INSERT OR IGNORE INTO pending_actions
     (id, tenant_id, proposed_at, proposed_by, capability_class, integration,
      action_type, state, authorization_level, send_delay_seconds,
      payload_r2_key, payload_hash, retry_count, max_retries)
     VALUES (?, ?, ?, ?, ?, ?, ?, 'pending', 'YELLOW', 0, ?, ?, 0, 3)`
  ).bind(
    msg.action_id, msg.tenant_id, now, msg.proposed_by,
    msg.capability_class, msg.integration, msg.tool_name,
    msg.payload_r2_key, msg.payload_hash
  ).run()

  // If INSERT was ignored (already processed), ack and skip
  if (inserted.meta.changes === 0) {
    rawMsg.ack()
    return
  }

  // Run authorization gate
  const authResult = await runAuthorizationGate(
    msg.tenant_id, msg.capability_class, msg.integration, env
  )

  // Handle HMAC failure — write anomaly signal and abort
  if (!authResult.hmacValid) {
    await writeAnomalyAndAudit(db, msg, 'action.hmac_invalid', now)
    await broadcastEvent(env, msg.tenant_id, {
      type: 'action.blocked', action_id: msg.action_id,
      tool_name: msg.tool_name, capability_class: msg.capability_class,
      tenant_id: msg.tenant_id, reason: 'hmac_invalid'
    })
    rawMsg.ack()
    return
  }

  // TOCTOU verification
  const hashValid = await verifyPayloadHash(msg.payload_stub, msg.payload_hash)
  if (!hashValid) {
    await writeAnomalyAndAudit(db, msg, 'action.toctou_violation', now)
    await broadcastEvent(env, msg.tenant_id, {
      type: 'action.toctou_violation', action_id: msg.action_id, tenant_id: msg.tenant_id
    })
    rawMsg.ack()
    return
  }

  // Route by effective authorization level
  switch (authResult.effectiveLevel) {
    case 'GREEN':
      await routeGreen(msg, authResult.sendDelaySeconds, env, now)
      break
    case 'YELLOW':
      await routeYellow(msg, authResult.sendDelaySeconds, env, now)
      break
    case 'RED':
      await routeRed(msg, env, now)
      break
  }

  rawMsg.ack()
}

async function routeGreen(msg: ActionQueueMessage, sendDelay: number, env: Env, now: number) {
  const db = env.D1_US
  const executeAfter = sendDelay > 0 ? now + sendDelay * 1000 : null

  await db.batch([
    db.prepare(
      `UPDATE pending_actions SET state = 'queued', authorization_level = 'GREEN',
       send_delay_seconds = ?, execute_after = ?, approved_by = 'auto_green', approved_at = ?
       WHERE id = ?`
    ).bind(sendDelay, executeAfter, now, msg.action_id),
    db.prepare(
      `INSERT INTO action_audit (id, tenant_id, action_id, created_at, event, agent_identity)
       VALUES (?, ?, ?, ?, 'action.routed_green', ?)`
    ).bind(crypto.randomUUID(), msg.tenant_id, msg.action_id, now, msg.proposed_by),
  ])

  if (executeAfter) {
    // TODO: Phase 2.x — replace with Workflow step.sleep() for durable send delay
    // For now: mark as queued with execute_after timestamp; Phase 1.4 Cron polls
    await db.prepare(
      `INSERT INTO action_audit (id, tenant_id, action_id, created_at, event, agent_identity)
       VALUES (?, ?, ?, ?, 'action.send_delay_started', ?)`
    ).bind(crypto.randomUUID(), msg.tenant_id, msg.action_id, now, msg.proposed_by).run()
    return  // Don't execute yet — Phase 1.4 picks up queued actions with execute_after
  }

  // Execute immediately (stub)
  await executeStub(msg, env, now)
}

async function routeYellow(msg: ActionQueueMessage, sendDelay: number, env: Env, now: number) {
  const db = env.D1_US
  const executeAfter = sendDelay > 0 ? now + sendDelay * 1000 : null

  await db.batch([
    db.prepare(
      `UPDATE pending_actions SET state = 'awaiting_approval',
       authorization_level = 'YELLOW', send_delay_seconds = ?, execute_after = ?
       WHERE id = ?`
    ).bind(sendDelay, executeAfter, msg.action_id),
    db.prepare(
      `INSERT INTO action_audit (id, tenant_id, action_id, created_at, event, agent_identity)
       VALUES (?, ?, ?, ?, 'action.routed_yellow', ?)`
    ).bind(crypto.randomUUID(), msg.tenant_id, msg.action_id, now, msg.proposed_by),
  ])

  await broadcastEvent(env, msg.tenant_id, {
    type: 'action.pending_approval', action_id: msg.action_id,
    tool_name: msg.tool_name, capability_class: msg.capability_class,
    tenant_id: msg.tenant_id
  })
}

async function routeRed(msg: ActionQueueMessage, env: Env, now: number) {
  const db = env.D1_US
  await db.batch([
    db.prepare(
      `UPDATE pending_actions SET state = 'rejected', authorization_level = 'RED',
       cancelled_at = ?, cancel_reason = 'hard_floor_red'
       WHERE id = ?`
    ).bind(now, msg.action_id),
    db.prepare(
      `INSERT INTO action_audit (id, tenant_id, action_id, created_at, event, agent_identity)
       VALUES (?, ?, ?, ?, 'action.routed_red', ?)`
    ).bind(crypto.randomUUID(), msg.tenant_id, msg.action_id, now, msg.proposed_by),
  ])

  await broadcastEvent(env, msg.tenant_id, {
    type: 'action.blocked', action_id: msg.action_id,
    tool_name: msg.tool_name, capability_class: msg.capability_class, tenant_id: msg.tenant_id
  })
}

async function executeStub(msg: ActionQueueMessage, env: Env, now: number) {
  const db = env.D1_US
  await db.batch([
    db.prepare(
      `UPDATE pending_actions SET state = 'completed', executed_at = ?,
       result_summary = 'stub_executed'
       WHERE id = ?`
    ).bind(now, msg.action_id),
    db.prepare(
      `INSERT INTO action_audit (id, tenant_id, action_id, created_at, event, agent_identity)
       VALUES (?, ?, ?, ?, 'action.executed_stub', ?)`
    ).bind(crypto.randomUUID(), msg.tenant_id, msg.action_id, now, msg.proposed_by),
  ])

  // TODO: Phase 2.1 — write episodic memory to Hindsight on successful execution
  // episodic_memory_id = await hindsightClient.retain({ type: 'action_executed', ... })

  await broadcastEvent(env, msg.tenant_id, {

    tool_name: msg.tool_name, tenant_id: msg.tenant_id

}

async function writeAnomalyAndAudit(db: D1Database, msg: ActionQueueMessage, event: string, now: number) {
  await db.batch([
    db.prepare(
      `UPDATE pending_actions SET state = 'failed', cancelled_at = ?, cancel_reason = ?
       WHERE id = ?`
    ).bind(now, event, msg.action_id),
    db.prepare(
      `INSERT INTO action_audit (id, tenant_id, action_id, created_at, event, agent_identity)
       VALUES (?, ?, ?, ?, ?, ?)`
    ).bind(crypto.randomUUID(), msg.tenant_id, msg.action_id, now, event, msg.proposed_by),
    db.prepare(
      `INSERT INTO anomaly_signals (id, tenant_id, created_at, signal_type, severity, related_id, detail_json)
       VALUES (?, ?, ?, ?, 'high', ?, ?)`
    ).bind(
      crypto.randomUUID(), msg.tenant_id, now,
      event === 'action.toctou_violation' ? 'toctou_violation' : 'authz_downgrade_attempt',
      msg.action_id,
      JSON.stringify({ action_id: msg.action_id, tool_name: msg.tool_name })
    ),
  ])
}

async function broadcastEvent(env: Env, tenantId: string, payload: unknown) {
  try {
    const id = env.MCPAGENT.idFromName(tenantId)
    const stub = env.MCPAGENT.get(id) as DurableObjectStub<any>
    // @ts-expect-error — DO RPC method (confirmed 1.2 pattern)
    await stub.broadcast(payload)
  } catch {
    // Non-fatal — WebSocket push is best-effort; Pages UI polls as fallback
  }
}

### Discovered Constraints
- **Cloudflare Queues don't have separate consumer Workers.** The queue consumer binding in `wrangler.toml` resolves to the `queue()` handler exported from the Worker's main entry point. No separate wrangler file needed.
- **Platform max_retries vs application max_retries are independent.** `wrangler.toml [[queues.consumers]] max_retries = 3` controls platform-level retries before DLQ routing. `pending_actions.max_retries = 3` is the application-level retry budget tracked in D1 code. Both set to 3 but operate at different layers.
- **tenants table column names must be verified against DDL.** Initial test helper used `jwt_sub` and `display_name` which don't exist — actual columns are `hindsight_tenant_id`, `data_region`, `primary_channel`, etc. per `1001_brain_tenants.sql`.
