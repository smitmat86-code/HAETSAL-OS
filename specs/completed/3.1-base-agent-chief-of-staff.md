## What We're Building

The cognitive foundation everything in Phase 3+ runs on. Two things:

**BaseAgent** — an abstract class that every domain agent extends. It implements
the full agent lifecycle: `open()` loads relevant memories from Hindsight to prime
context, `run()` executes the agent loop with doom loop detection and context
budget monitoring, `close()` writes session synthesis as an episodic memory and
encrypts the reasoning trace. Law 3 is enforced structurally: `BaseAgent.retain()`
only accepts `'episodic' | 'semantic' | 'world'` — procedural writes throw at
compile time, not runtime.

**Chief of Staff** — the first concrete agent built on BaseAgent. Layer 2
orchestrator: receives requests from the Layer 1 router, understands the tenant's
full context (active goals, pending actions, recent decisions), and either handles
the request directly or delegates to a domain agent. Maintains `parent_trace_id`
chains so every agent decision is traceable back to the triggering request.

**Layer 1 Router** — lightweight pattern matcher that sits in front of all agents.
Classifies incoming requests by pattern first (~5ms), falls back to Workers AI
classifier if ambiguous (~200ms). Routes to Chief of Staff, directly to a domain
agent, or handles trivially inline.

When complete: an MCP session with THE Brain routes through the Layer 1 Router →
Chief of Staff → domain agent (or handles directly), with full memory context
loaded, doom loop protection active, and session synthesis written at close.

## THE Brain Laws Check

### Law 1: One Public Face
No new HTTP surface. All agent execution happens inside McpAgentDO — the existing
MCP session path. Layer 1 Router is called from within the DO's tool handler, not
from a new route.

### Law 2: Zero-Knowledge
- Reasoning traces encrypted with TMK before storage (R2_OBSERVABILITY)
- Memory loads from Hindsight: content decrypted in DO using TMK before use
- Session synthesis: plaintext assembled in DO, encrypted via `retainContent()`
  before Hindsight retain
- `parent_trace_id` chains: UUIDs only — no content in trace metadata

### Law 3: Agent Write Policy (STRUCTURAL ENFORCEMENT)
```typescript
// BaseAgent.retain() — the ONLY memory write path for agents
// Structurally prevents procedural writes at the type level

protected async retain(
  content: string,
  memoryType: EpistemicMemoryType,  // 'episodic' | 'semantic' | 'world' only
  domain: string,
  provenance: string,
): Promise<RetainResult | null>

// EpistemicMemoryType deliberately excludes 'procedural'
// Procedural writes ONLY happen in 3.3 consolidation cron — not through BaseAgent
type EpistemicMemoryType = 'episodic' | 'semantic' | 'world'
```

This is structural, not policy. An agent cannot write procedural memories even if
it tries — the type doesn't exist in BaseAgent's retain signature.

## BaseAgent Lifecycle

```
open()
  ├── Load mental model (domain-specific semantic memories, mode='default')
  ├── Load recent episodic context (last 7 days, mode='timeline' stub → default)
  ├── Load active pending_actions for context (D1 query)
  └── Prime agent context with decrypted memories

run(input)
  ├── Layer 1 Router classifies request
  ├── Agent loop:
  │   ├── Call AI model (Workers AI via AI Gateway)
  │   ├── Track token usage (input_tokens + output_tokens from response)
  │   ├── Check context budget (cumulative_tokens / model_limit > 0.8 → flush)
  │   ├── Execute tool calls
  │   ├── Doom loop check (same tool + same input_hash → warn/break)
  │   └── Repeat until done or circuit break
  └── Return response

close()
  ├── Write session synthesis (episodic memory of what happened)
  ├── Encrypt reasoning trace → R2_OBSERVABILITY
  └── Clear in-session state
```

### `open()` — Memory Load

```typescript
protected async open(): Promise<void> {
  const tmk = this.env.getMcpAgentTmk()  // already in DO memory
  const hTenantId = this.hindsightTenantId

  // Load domain mental model
  const mentalModelResults = await this.hindsightRecall({
    tenant_id: hTenantId,
    query_encrypted: await this.encryptQuery(`${this.domain} mental model goals context`),
    domain: this.domain,
    limit: 10,
  })

  // Load recent episodic context
  const episodicResults = await this.hindsightRecall({
    tenant_id: hTenantId,
    query_encrypted: await this.encryptQuery(`recent events decisions ${this.domain}`),
    domain: this.domain,
    limit: 5,
  })

  // Decrypt and prime context
  this.primeContext([...mentalModelResults, ...episodicResults], tmk)

  // Load active pending_actions (unencrypted metadata only — Law 2 compliant)
  const pendingActions = await this.env.D1_US.prepare(
    `SELECT id, tool_name, integration, capability_class, state, proposed_at
     FROM pending_actions
     WHERE tenant_id = ? AND state IN ('awaiting_approval', 'queued')
     ORDER BY proposed_at DESC LIMIT 5`
  ).bind(this.tenantId).all()

  this.context.pendingActions = pendingActions.results
}
```

**Recall response handling — no `occurred_at` on results:**
Results are unordered by time. BaseAgent accepts this for Phase 3 — memories are
loaded by relevance (confidence + relevance scores), not chronologically.
Phase 4.3 adds temporal ordering when confidence propagation is built.

**Recall response handling — `deferred` retain status:**
When `retainContent()` returns a result with a deferred Hindsight status, treat
as success — the memory will be eventually consistent. Do not retry.

### `run()` — Agent Loop with Safety Primitives

```typescript
// Doom loop detector state (per run() invocation)
interface DoomLoopState {
  calls: Array<{ toolName: string; inputHash: string }>
  warnCount: number
}

function computeInputHash(toolName: string, input: unknown): string {
  // SHA-256 of toolName + JSON.stringify(input)
  // Same tool + same input = same hash = doom loop candidate
}

function checkDoomLoop(state: DoomLoopState, toolName: string, input: unknown): 'ok' | 'warn' | 'break' {
  const hash = computeInputHash(toolName, input)
  const recentSame = state.calls.filter(c => c.toolName === toolName && c.inputHash === hash)

  if (recentSame.length >= 5) return 'break'   // Circuit break — hard stop
  if (recentSame.length >= 3) {
    state.warnCount++
    return 'warn'                               // Log warning, inject correction prompt
  }
  state.calls.push({ toolName, inputHash: hash })
  return 'ok'
}

// Context budget check
// Workers AI response includes usage: { input_tokens, output_tokens }
// MODEL_CONTEXT_LIMIT: 128_000 for llama-3.3-70b-instruct-fp8-fast
const MODEL_CONTEXT_LIMIT = 128_000
const FLUSH_THRESHOLD = 0.80

function shouldFlush(cumulativeTokens: number): boolean {
  return cumulativeTokens / MODEL_CONTEXT_LIMIT > FLUSH_THRESHOLD
}

// On flush: summarize conversation so far, reset context, continue
// Flush writes an intermediate episodic memory: "Session in progress: [summary]"
```

**Doom loop warning injection:**
When `warn` triggered, inject into next model call:
```
"[SYSTEM: You have called {toolName} with identical inputs {n} times.
Consider a different approach or ask the user for clarification.]"
```

**Circuit break behavior:**
Hard stop. Write anomaly signal to `anomaly_signals` table. Return to user:
`"I appear to be stuck in a loop on this request. Could you rephrase or clarify?"`
Do NOT continue the agent loop.

### `close()` — Session Synthesis

```typescript
protected async close(sessionSummary: string): Promise<void> {
  // 1. Write session synthesis as episodic memory
  await this.retain(
    sessionSummary,
    'episodic',
    this.domain,
    'agent_session'
  )

  // 2. Encrypt reasoning trace → R2_OBSERVABILITY
  const traceKey = `traces/${this.tenantId}/${this.traceId}`
  const encryptedTrace = await this.encryptContent(JSON.stringify(this.reasoningTrace))
  // waitUntil not available in DO — fire and move on
  // R2 write may fail silently — non-critical
  this.env.R2_OBSERVABILITY.put(traceKey, encryptedTrace).catch(() => {})

  // 3. Clear session state
  this.context = this.freshContext()
}
```

## Layer 1 Router

```typescript
// src/services/agents/router.ts (~80 lines)
// Pattern-first, classifier fallback — same principle as write-policy validator

const ROUTING_PATTERNS: Array<{ pattern: RegExp; agent: AgentType }> = [
  { pattern: /\b(career|work|job|project|deadline|client|promotion)\b/i, agent: 'career_coach' },
  { pattern: /\b(workout|health|sleep|exercise|doctor|fitness)\b/i, agent: 'life_coach' },
  { pattern: /\b(budget|invoice|expense|payment|salary|finance)\b/i, agent: 'career_coach' },
  { pattern: /\b(relationship|friend|family|partner)\b/i, agent: 'chief_of_staff' },
  { pattern: /\b(remind|schedule|calendar|meeting|plan)\b/i, agent: 'chief_of_staff' },
]

type AgentType = 'chief_of_staff' | 'career_coach' | 'life_coach' | 'inline'

export async function routeRequest(
  input: string,
  env: Env
): Promise<AgentType> {
  // Pattern check — fast
  for (const { pattern, agent } of ROUTING_PATTERNS) {
    if (pattern.test(input)) return agent
  }

  // Ambiguous — Workers AI classifier via AI Gateway
  const result = await env.AI.run(
    '@cf/meta/llama-3.1-8b-instruct',
    {
      messages: [{
        role: 'user',
        content: `Classify this request into one category:
chief_of_staff (general planning, orchestration, multi-domain)
career_coach (work, projects, professional goals)
life_coach (health, relationships, personal growth)
inline (simple factual question, no agent needed)

Request: "${input.slice(0, 300)}"
Answer with exactly one category name.`
      }]
    },
    { gateway: { id: 'brain-gateway' } }
  ) as { response: string }

  const classification = result.response.trim().toLowerCase()
  if (['chief_of_staff', 'career_coach', 'life_coach', 'inline'].includes(classification)) {
    return classification as AgentType
  }
  return 'chief_of_staff'  // Default fallback
}
```

## Chief of Staff

```typescript
// src/agents/chief-of-staff.ts (~120 lines)
// Layer 2 orchestrator — knows the full picture, delegates or handles

export class ChiefOfStaff extends BaseAgent {
  readonly domain = 'general'
  readonly agentIdentity = 'chief_of_staff'

  protected systemPrompt(): string {
    return `You are THE Brain's Chief of Staff — the tenant's most trusted advisor.
You have full context of their goals, pending decisions, and active projects.
You orchestrate other agents, propose actions, and maintain strategic continuity.

Current context (loaded at session open):
${this.contextSummary()}

Pending actions awaiting approval: ${this.context.pendingActions.length}
Active domains: ${this.activeDomains().join(', ')}

You can:
- Answer questions directly using loaded memory context
- Propose actions via brain_v1_act_* tools (they go through authorization gate)
- Delegate to domain agents by signaling agent_type in your response
- Surface connections between domains that the tenant may not have noticed

You cannot:
- Write procedural memories (only the nightly cron can)
- Access raw action payloads (Law 2)
- Bypass the authorization gate`
  }

  async run(input: string, traceId: string, parentTraceId?: string): Promise<string> {
    this.traceId = traceId
    this.parentTraceId = parentTraceId  // Chain for causal tracing

    await this.open()

    const response = await this.agentLoop(input)

    const synthesis = `Chief of Staff session: ${input.slice(0, 100)}... → ${response.slice(0, 100)}...`
    await this.close(synthesis)

    return response
  }
}
```

**`parent_trace_id` chaining:**
Every agent call includes a `traceId` (new UUID per session) and optionally a
`parentTraceId` (the traceId of the agent that triggered this one). This creates
a causal chain readable in Phase 5.2's reasoning trace viewer.

**Chief of Staff → domain agent delegation:**
In Phase 3.1, delegation is a signal in the response — the Chief of Staff returns
a structured response indicating which domain agent should handle follow-up. Full
agent-spawning (Chief of Staff programmatically invoking Career Coach) is Phase 3.2+.

## Cron KEK Provisioning (Wire in this session)

The nightly cron (3.3) needs a KEK to read memories. KEK is provisioned during
an active MCP session and stored in KV with 24h expiry.

```typescript
// Add to McpAgent.ts — called during session init (after TMK derivation)
async provisionCronKek(): Promise<void> {
  // Generate a new KEK for the cron
  const kek = crypto.getRandomValues(new Uint8Array(32))

  // Encrypt KEK with tenant TMK
  const tmk = this.getTmk()
  if (!tmk) return  // No active session — skip

  const iv = crypto.getRandomValues(new Uint8Array(12))
  const encrypted = await crypto.subtle.encrypt({ name: 'AES-GCM', iv }, tmk, kek)
  const cronKekEncrypted = btoa(
    String.fromCharCode(...new Uint8Array([...iv, ...new Uint8Array(encrypted)]))
  )

  // Store in KV (24h expiry) + update tenants table
  const expiresAt = Date.now() + 24 * 60 * 60 * 1000
  await this.env.KV_SESSION.put(
    `cron_kek:${this.tenantId}`,
    cronKekEncrypted,
    { expirationTtl: 86400 }
  )
  await this.env.D1_US.prepare(
    `UPDATE tenants SET cron_kek_encrypted = ?, cron_kek_expires_at = ? WHERE id = ?`
  ).bind(cronKekEncrypted, expiresAt, this.tenantId).run()
}
```

This runs once per session — idempotent (overwrites previous KEK with fresh one).
If the tenant hasn't opened a session in 24h, cron KEK is expired and the nightly
cron defers (does not fail — defers to next session).
