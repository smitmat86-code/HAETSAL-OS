# Spec 3.4 — Morning Brief + Predictive Heartbeat + Weekly Synthesis

## Goal

Build THE Brain's proactive delivery layer — a rich daily morning brief (7 AM), a lightweight heartbeat that pings only when actionable (every 30 min, 8AM–8PM), and a weekly synthesis (Friday 5 PM). Primary delivery: Telegram bot. Secondary: Pages UI WebSocket push + Obsidian.

---

## User Review Required

> [!WARNING]
> **New external dependencies introduced:**
> 1. **Telegram Bot API** — requires BotFather setup and `TELEGRAM_BOT_TOKEN` + `TELEGRAM_WEBHOOK_SECRET` secrets
> 2. **Brave Search API** — requires API key (`BRAVE_API_KEY` secret)
> 3. **bible-api.com** — public, no key needed, KJV translation
>
> These are 3 new secrets to provision before deployment. Phase 3.4 uses manual Telegram chat_id setup (tenant enters via settings page), not automated /start flow.

> [!IMPORTANT]
> **Law 1 exception**: `/telegram/webhook` route is NOT behind CF Access — Telegram servers must reach it directly. Validated via `X-Telegram-Bot-Api-Secret-Token` header instead. This is the same pattern used for the Telnyx SMS webhook (spec 2.1 precedent).

**Post-delivery:**
- Archive to Hindsight as episodic memory (encrypted with KEK via `retainContent()`)

### Predictive Heartbeat

#### [NEW] [heartbeat.ts](file:///c:/Users/matth/Documents/HAETSAL%20OS/src/cron/heartbeat.ts)

30-minute background check (~100 lines):

- `runPredictiveHeartbeat(env, ctx)` — exported, called from `scheduled()`
- **Time window**: `hour < 8 || hour >= 20` → return immediately (UTC)
- Per tenant with `bootstrap_status = 'completed'`:

**Alert checks:**
1. Pending actions > 20h old, still `awaiting_approval` → "⏰ Actions expiring soon"
2. > 2 unsurfaced high-priority consolidation gaps → "💭 N open questions"
3. No session today (Phase 4+ — TODO stub)

- **Only sends if alerts exist.** No-alert path = total silence. This is correct behavior.
- Telegram-only delivery (not Pages or Obsidian)

### Source Modifications

#### [MODIFY] [index.ts](file:///c:/Users/matth/Documents/HAETSAL%20OS/src/workers/mcpagent/index.ts)

1. Add to `scheduled()` switch:
```typescript
case '0 7 * * *':   return handleMorningBrief(env, ctx)
case '*/30 * * * *': return runPredictiveHeartbeat(env, ctx)
case '0 17 * * 5':   return runWeeklySynthesis(env, ctx)
```

> [!NOTE]
> **Cron overlap at 7:00 AM / 7:30 AM**: The morning brief (`0 7 * * *`) and heartbeat (`*/30 * * * *`) both fire at 7:00 AM and 7:30 AM. This is safe: Cloudflare fires each cron expression as a **separate** `scheduled()` invocation — they don't share a call. The heartbeat's time window check (`hour < 8`) means it returns immediately at 7:00 and 7:30 AM anyway. No conflict, but worth documenting so nobody panics seeing both fire.

2. Add `/telegram/webhook` route **before** auth middleware (same pattern as SMS ingest):
```typescript
app.post('/telegram/webhook', async (c) => {
  const secretToken = c.req.header('X-Telegram-Bot-Api-Secret-Token')
  if (secretToken !== c.env.TELEGRAM_WEBHOOK_SECRET) return c.json({}, 403)
  // Phase 3.4: acknowledge only — full /start flow is Phase 5+
  return c.json({ ok: true })
})
```

3. Add `POST /api/settings/telegram` route (behind auth):
```typescript
// Manual chat_id setup for Phase 3.4
app.post('/api/settings/telegram', async (c) => {
  const { chat_id } = await c.req.json()
  await c.env.KV_SESSION.put(`telegram_chat_id:${c.get('tenantId')}`, String(chat_id))
  return c.json({ ok: true })
})
```

## Constraints Applied

| Constraint | Enforcement |
|-----------|------------|
| `morning-brief.ts` ≤ 180 lines | Section fetchers are concise helper functions |
| `heartbeat.ts` ≤ 120 lines | D1 queries + conditional send only |
| `telegram.ts` ≤ 100 lines | Send + webhook only |
| `Promise.allSettled()` for brief sections | Each section fails independently |
| KV cache for Bible verse (24h) | Do not call bible-api.com more than once/day |
| `generated_by: the-brain` frontmatter | Anti-circular on all Obsidian output |
| KEK from `kek.ts` service | Do not re-implement inline |
| Telegram webhook before auth middleware | Same pattern as Telnyx SMS |
| No heartbeat outside 8-20 UTC | Early return check |
| `ctx.waitUntil()` for Hindsight archive | Non-blocking |
