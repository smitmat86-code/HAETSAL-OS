# Spec 3.3 v2 — Nightly Consolidation Cron (Post-Hindsight Audit)

## Goal

Build THE Brain's "sleep cycle" — now **4 passes** (down from 6). Hindsight natively handles fact distillation and mental model synthesis. Our cron focuses on genuinely custom work: contradiction detection, bridge edges, behavioral patterns, and gap preparation. **Webhook-driven** with cron fallback.

---

## What Changed From v1 Plan

| v1 Pass | v2 Status | Why |
|---------|----------|-----|
| Pass 1 — Fact Distillation | **REMOVED** | Hindsight does this via `/consolidate` + `observations_mission` |
| Pass 2 — Contradiction Resolution | **KEPT** — now Pass 1 | Uses `/memories/{id}/history` for structural signal before LLM |
| Pass 3 — Bridge Edges | **KEPT** — now Pass 2 | Seeded from `/graph` endpoint (structural, no decrypt) |
| Pass 4 — Behavioral Patterns | **KEPT** — now Pass 3 | Unchanged — sole procedural write path |
| Pass 5 — Mental Models | **REMOVED** | Hindsight Mental Models API with `refresh_after_consolidation: true` |
| Pass 6 — Gap Questions | **KEPT** — now Pass 4 | Uses `/reflect` with `response_schema` (structured JSON) |

**New: Webhook-driven trigger.** Hindsight fires `consolidation.completed` → our `/hindsight/webhook` route → `runConsolidationPasses()`. The `0 2 * * *` cron is fallback only.

**New: `BaseAgent.open()` change.** Mental models loaded via Hindsight's `/mental-models/{id}` (plaintext) instead of recall + metadata filter.

### Consolidation Orchestrator

#### [NEW] [consolidation.ts](file:///c:/Users/matth/Documents/HAETSAL%20OS/src/cron/consolidation.ts) (~160 lines)

**Two entry points** (new in v2):
1. `runConsolidationPasses(tenantId, env, ctx)` — webhook-triggered (preferred)
2. `handleNightlyConsolidation(env, ctx)` — cron fallback, iterates all tenants

Both share inner logic:
1. Fetch + validate KEK → defer if expired/missing
2. `INSERT OR IGNORE` consolidation_runs row (dedup by day)
3. Pass 1→2→3→4 **sequentially** (each `await`ed)
4. Update run row with per-pass counts, status='completed'
5. On error: status='failed', error_message

### 4 Pass Modules

#### [NEW] [pass1-contradiction.ts](file:///c:/Users/matth/Documents/HAETSAL%20OS/src/cron/passes/pass1-contradiction.ts) (~80 lines)

**v2 change**: Uses Hindsight `/memories/list` + `/memories/{id}/history` (structural signal before LLM):
- List semantic memories (limit 50)
- For each: check `/history` for 2+ entries with significant content change
- Batch candidates into single LLM call for resolution classification
- `genuine_contradiction` → anomaly_signal. `ambiguous` → anomaly_signal. `natural_update` → no action
- No decryption needed for structural check; decrypt only if LLM needs content

#### [NEW] [pass2-bridges.ts](file:///c:/Users/matth/Documents/HAETSAL%20OS/src/cron/passes/pass2-bridges.ts) (~80 lines)

**v2 change**: Seeded from `/graph` endpoint (fast, structural, no decrypt):
- Fetch `/graph?limit=200` → nodes + edges
- Find structural holes: cross-domain node pairs sharing neighbors but no direct edge
- Top 10 candidates by shared-neighbor count
- Decrypt only candidate pair content with KEK for LLM insight generation
- Retain bridges as semantic with `{ is_bridge: true, bridge_memory_ids: [...] }` metadata
- Max 5 bridges

#### [NEW] [pass3-patterns.ts](file:///c:/Users/matth/Documents/HAETSAL%20OS/src/cron/passes/pass3-patterns.ts) (~90 lines)

**Unchanged from v1** — sole procedural write path:
- Recall episodic (30 days, session provenance, limit 50), decrypt with KEK
- LLM: extract behavioral patterns (confidence > 0.6, max 3)
- `consolidationRetain(pattern, 'procedural', domain, 'pass3_behavioral')` — module-private, NOT exported

#### [NEW] [pass4-gaps.ts](file:///c:/Users/matth/Documents/HAETSAL%20OS/src/cron/passes/pass4-gaps.ts) (~70 lines)

**v2 change**: Uses Hindsight `/reflect` with `response_schema`:
- POST to `/reflect` with structured JSON schema requiring `{ gaps: [{ question, domain, priority }] }`
- Hindsight uses full retrieval stack (semantic + BM25 + entity graph + temporal) — better signal than raw LLM
- Top 3 gaps written to `consolidation_gaps` D1 table (metadata, not content)
- No encryption on gap questions — structural only

## Constraints Applied

| Constraint | Enforcement |
|-----------|------------|
| `consolidation.ts` ≤ 180 lines | Pass logic in separate modules |
| `pass3-patterns.ts` ≤ 100 lines | Focused: recall + LLM + procedural retain |
| `kek.ts` ≤ 80 lines | Fetch + validate + crypto only |
| Migration ≤ 50 lines | 2 tables + 3 indexes |
| Passes sequential 1→4 | Each `await`ed before next |
| `Promise.allSettled()` per-item | Within each pass, items in parallel |
| `consolidationRetain()` NOT exported | Module-private in pass3-patterns.ts |
| AI Gateway on all LLM calls | `{ gateway: { id: 'brain-gateway' } }` |
| `/graph` for bridge seeding | NOT recall — structural, no decrypt |
| `/reflect` with `response_schema` for gaps | NOT raw LLM — Hindsight full retrieval |
| Mental Model content is plaintext | No KEK needed to read |
| Webhook before auth middleware | Same precedent as Telnyx/Telegram |
| Dedup unique index | Prevents webhook + cron double-run same night |
